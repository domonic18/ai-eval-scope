### 1.2 三字段定义与消费

## 二、核心原则：字段 × 数据源分策略

**不同字段适合不同数据源**——这是四轮试点验证的核心结论，不能"一个数据源搞定所有字段"。

| 字段 | 最优数据源类型 | 核心优势 |
|------|----------------|----------|
| **constants** | **结构化数值表** | value 权威准确、数值密集 |
| **misconceptions** | **评测选择题** | 干扰项=现成误解、即生即用 |
| **domain_facts** | **按类型分源** | 公式库/诗词/地理/历史各有最优源 |

## 三、试点验证结论（核心证据）

### 3.1 验证过程

本项目通过四轮试点验证了不同字段 × 数据源的适配性。验证分两阶段：
- **阶段 1**（索引更新）：调研 opencompass 167 个数据集，更新 `dataset_index.yaml`（13→14 个，新增 openbookqa + `knowledge_mapping` 字段）
- **阶段 2**（下载 + 提取试点）：下载 arc / cmmlu / 周期表 JSON，分别用 LLM 提取和结构化转换验证三字段产出

### 3.2 四轮试点数据

| 试点 | 数据源 | 字段 | 产出 | value 准确性 | 结论 |
|------|--------|------|------|-------------|------|
| ① arc 20 题 | 评测题（通用科学） | constants | 2 条（10%） | ❌ 分数陷阱（1/6 提取成 1） | 评测题 constants ROI 低 |
| ② cmmlu 物理/化学 30 题 | 评测题（学科子集筛选） | constants | 1 条（3.3%） | ❌ LLM regex 截断 | 筛选学科未提升（概念题数值稀疏） |
| ③ **周期表 JSON 30 元素** | **结构化数值表** | constants | **116 条（386%）** | ✅ **100% 权威标准值** | **结构化源完胜**（ROI 提升 100 倍） |
| ④ **arc 20 题** | **评测题** | misconceptions | **58 条（290%）** | ✅ 100% 即生即用 | **评测题 misconceptions ROI 高** |

### 3.3 关键结论

1. **constants 的评测题驱动不可行**：评测题考"理解/判断"，数值只是少数题的背景信息。筛选学科子集（cmmlu 物理 3.3% < arc 10%）**未提高 ROI**——学科对了但题型仍是概念题。**constants 的权威源是结构化数值表**（教材/物理手册/周期表），不是评测题。
2. **constants 的结构化源驱动最优**：周期表 JSON 每元素 4-5 个数值属性，ROI 是评测题的 38-100 倍，value 全部权威（零 LLM 猜测）。
3. **misconceptions 的评测题驱动最优**：选择题干扰项天然对齐"误解"，ROI 290%，pattern 用关键词即可（无需人工 regex）。

### 3.4 opencompass 数据集与 K-12 适配性（评估记录）

opencompass 定位大模型通用能力评测，多为**通用能力/高中-大学级**题目，而本项目 knowledge 聚焦**中小学学科知识点**——年级错位是核心约束：

| 数据集 | 类型 | 与 K-12 对应 | 可提取字段 |
|--------|------|--------------|-----------|
| arc | 中小学科学 | **高**（primary/junior） | misconceptions ✅ |
| cmmlu / ceval / mmlu | 多学科 | 中（senior/college） | misconceptions（学段偏高） |
| openbookqa | 小学科学 | 高（primary） | misconceptions ✅ |
| gsm8k / mbpp / humaneval / bbh | 能力评测 | 低 | ❌ 非学科知识点 |

> **结论**：opencompass 数据集用于 misconceptions（干扰项提取）可行；用于 constants（数值事实）不可行（数值稀疏）。

## 四、constants 更新策略（结构化源驱动）

### 4.1 提取流程

```
结构化数值表（周期表 JSON / NIST CODATA / Wikidata SPARQL）
  → value/tolerance 直接取（权威准确，零 LLM 猜测）
  → extract_pattern：模板生成 → 真实课件文本验证迭代
  → 审核合并 knowledge yaml
```

### 4.2 数据源（按学科）

| 学科 | 数据源 | 覆盖 | 格式 | 状态 |
|------|--------|------|------|------|
| **chemistry** | **[周期表 JSON（Bowserinator）](https://github.com/Bowserinator/Periodic-Table-JSON)** | 119 元素 × 沸点/熔点/原子量/密度 | JSON | ✅ **已验证（116 条）** |
| **physics** | **[NIST CODATA SRD-121](https://data.nist.gov/pdr/lps/FDB5909746705200E043065706813E54120)** | 基本常数（g/c/h/普朗克） | JSON | 待验证 |
| **physics** | **[Wikidata SPARQL](https://query.wikidata.org/)** | 水沸点、光速等非基本常数 | SPARQL→JSON | 待验证 |
| biology | Wikidata SPARQL | 人体常数（体温 37℃、心率等） | SPARQL | 待验证 |
| geography | 手工/Wikidata | 地球半径、赤道长度等 | 手工 | 低优先 |

### 4.3 执行步骤（以 chemistry 为例，已验证路径）

1. 下载周期表 JSON：`curl -L https://raw.githubusercontent.com/Bowserinator/Periodic-Table-JSON/master/PeriodicTableJSON.json`
2. 转换：符号→中文名映射 + Kelvin→℃ + 生成 `{name, value, tolerance, extract_pattern}`（**待固化为 `scripts/convert_periodic_table.py`**）
3. regex 课件验证：用 116 条 `extract_pattern` 在真实化学课件上跑，检验命中率
4. 审核 + 合并 `chemistry.yaml`（constants 3→~100）

### 4.4 extract_pattern 验证标准

- 在真实课件文本中**命中率 > 70%**（能提取出 value 附近的数字）
- **误匹配率 < 5%**（不抓取无关数字）
- 不达标 → 人工修正 regex → 重验

## 五、misconceptions 更新策略（评测题驱动）

### 5.1 提取流程

```
评测选择题（arc/cmmlu/ceval/mmlu）
  → 每题 3 个干扰项 = 3 个误解候选
  → LLM 提取 {pattern(关键词), correct(正确答案), description, severity}
  → pattern 用关键词（非复杂 regex），即生即用
  → 审核合并 knowledge yaml
```

### 5.2 数据源（按学科适配性）

| 数据集 | 类型 | K-12 适配 | 学科映射 | 试点 |
|--------|------|-----------|----------|------|
| **arc** | 中小学科学 | ⭐ 最贴合（primary/junior） | physics/chemistry/biology | ✅ 58 条 |
| **cmmlu** | 中文多学科（67 学科） | 中（senior/college） | 全学科 | 数据集已下载 |
| ceval | 中文考试（52 学科） | 中（senior/college） | 全学科 | 索引已收录 |
| mmlu | 英文多学科 | 中（英文） | 全学科 | 索引已收录 |
| openbookqa | 小学科学 | ⭐ 高（primary） | science | 索引已收录 |

### 5.3 执行步骤

1. 解析：`python scripts/parse_dataset.py --dataset arc --output questions.jsonl --limit N`
2. LLM 提取：`python scripts/extract_knowledge.py --field misconceptions --questions ... --output ...`
3. 人工抽审（pattern 准确性、correct 科学性）
4. 按学科合并 knowledge（arc→physics/chemistry/biology）

### 5.4 pattern 质量标准

- **关键词匹配**（非复杂 regex），容错性高
- pattern 能匹配课件中该误解的典型表述
- correct 科学准确（对照教材/权威源）

## 六、domain_facts 更新策略（按类型分源）

### 6.1 前提：消费扩展先行

> domain_facts 当前仅 `math_formulas` 被代码消费（MathFormulaEvaluator）。**补数据前需先扩展消费**（让 LLM Judge 参考 domain_facts），否则 ROI 打折。

### 6.2 按类型的数据源（6 类）

#### 类型 1：公式类（math_formulas / physics_formulas）— 直接消费，最高价值

| 数据源 | 覆盖 | 格式 | K-12 适配 |
|--------|------|------|-----------|
| **[cn-k12（HuggingFace）](https://huggingface.co/datasets/aslawliet/cn-k12)** | K-12 数学题 + LaTeX 公式 | JSON | ✅ |
| **[tal-tech/chinese-k12-evaluation](https://github.com/tal-tech/chinese-k12-evaluation)** | K-12 知识图谱（500+ 一级、1900+ 二级知识点） | JSON | ✅ |
| **[CMATH（OpenDataLab）](https://opendatalab.com/OpenDataLab/CMATH)** | 中文数学数据集 | 多格式 | ✅ |
| 教材附录公式表 | 中学数学/物理公式 | 手工/OCR | ✅ 最贴合 |

**转换目标**：`math_formulas.{plane_figures, solid_figures, algebra_formulas}` 的 `{name, formula, description}` schema。

#### 类型 2：文学类（classical_poetry / common_idioms）

| 数据源 | 覆盖 | 格式 |
|--------|------|------|
| **[chinese-poetry/chinese-poetry](https://github.com/chinese-poetry/chinese-poetry)** | **5.5 万唐诗 + 26 万宋诗 + 2.1 万宋词** | JSON |
| 成语数据集（GitHub 多个项目） | 常用成语 + 释义 | JSON |

#### 类型 3：地理类（provincial_capitals / rivers / mountains）

| 数据源 | 覆盖 | 格式 |
|--------|------|------|
| **[uiwjs/province-city-china](https://github.com/uiwjs/province-city-china)** | 省/市/县/街道（GB/T 2260 国标） | JSON/CSV/SQL |
| **[Simplemaps China Cities](https://simplemaps.com/data/cn-cities)** | 461 城市（经纬度/人口） | CSV/JSON |

#### 类型 4：历史类（dynasty_timeline / historical_figures / key_events）

| 数据源 | 覆盖 | 格式 |
|--------|------|------|
| **[Wikidata SPARQL](https://query.wikidata.org/)** | 朝代/人物/事件（结构化） | SPARQL→JSON |

#### 类型 5：分类/属性类（common_elements / classification）

| 数据源 | 状态 |
|--------|------|
| **周期表 JSON**（已用于 constants） | ✅ 可复用补 `common_elements` |
| **Wikidata** | biology 分类（动物纲目） |

#### 类型 6：常识类（national_symbols / key_dates）

| 数据源 | 说明 |
|--------|------|
| 手工 / Wikidata | 量小，手工即可 |

### 6.3 执行步骤（公式类示例）

1. 下载 cn-k12 / tal-tech 数据
2. 筛选公式 → 转换为 `{name, formula, description}`
3. 验证：MathFormulaEvaluator 跑样本，确认公式校验生效
4. 合并 `math.yaml` domain_facts.math_formulas

## 九、审核与合并规范

### 9.1 审核标准

| 字段 | 审核要点 |
|------|----------|
| constants | value 准确（对照权威源）、extract_pattern 命中率 >70%、tolerance 合理 |
| misconceptions | pattern 能匹配典型表述、correct 科学准确、severity 合理 |
| domain_facts | 结构符合学科 schema、数据准确、source 可追溯 |

### 9.2 合并规范

- 每条新增项标注 `source`（数据源 + 原始 id），便于追溯与回滚
- 按 `source` 去重（避免同一知识点重复）
- 合并前跑 `make check`（ruff + pytest）确保不破坏
