# 端到端验收指南

> CLI 完整流程使用说明：打包 → 评估 → 查看报告。

---

## 一、完整流程

```bash
# 1. 打包
uv run agent-eval pack --source-dir /path/to/your/output/

# 2. 评估
uv run agent-eval eval \
  --package-dir ./packages/{目录名}/ \
  --rule-set ./assets/rules/default_rule_set.yaml
```

两步完成。不配置 LLM 时，14 项评估器中 9 项 Rule-based 正常运行，5 项 LLM 评估器自动降级（默认分 0.7）。

---

## 二、打包（pack）

将 Agent 产出物打包为标准 ExecutionPackage。

```bash
# 指定目录（自动遍历，task-id 取目录名）
uv run agent-eval pack --source-dir /path/to/your/output/

# 指定文件
uv run agent-eval pack --files doc1.md --files doc2.html

# 自定义任务信息
uv run agent-eval pack \
  --source-dir /path/to/your/output/ \
  --task-id math_001 \
  --task-title "一元一次方程" \
  --task-subject math

# 打包后验证
uv run agent-eval pack --source-dir /path/to/your/output/ --validate
```

| 参数 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `--files` | 二选一 | — | 文件路径（可多次指定） |
| `--source-dir` | 二选一 | — | 源目录路径 |
| `--task-id` | ❌ | 自动推导 | directory 取目录名，files 取时间戳 |
| `--task-title` | ❌ | 取 task-id | 任务标题 |
| `--task-subject` | ❌ | — | 任务学科（影响知识库加载） |
| `--output-dir` | ❌ | `./packages` | 输出目录 |
| `--validate` | ❌ | false | 打包后验证完整性 |

---

## 三、评估（eval）

对 ExecutionPackage 执行三阶段级联评估（格式 → 常识 → 质量）。

```bash
# 最简模式（无 LLM，快速验证）
uv run agent-eval eval \
  --package-dir ./packages/math_001 \
  --rule-set ./assets/rules/default_rule_set.yaml

# 含 LLM Judge
uv run agent-eval eval \
  --package-dir ./packages/math_001 \
  --rule-set ./assets/rules/default_rule_set.yaml \
  --llm-config ./llm_config.yaml \
  --llm-provider deepseek_judge \
  --verbose
```

| 参数 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `--package-dir` | ✅ | — | ExecutionPackage 目录路径 |
| `--rule-set` | ✅ | — | 规则集 YAML 文件路径 |
| `--output-dir` | ❌ | `./workspace` | 输出目录 |
| `--llm-config` | ❌ | — | LLM 配置文件路径 |
| `--llm-provider` | ❌ | 默认 provider | 覆盖 LLM Provider 名称 |
| `--project` | ❌ | — | 项目 ID |
| `--eval-mode` | ❌ | `pipeline` | 评估模式（`agent` 未实现） |
| `--verbose / -v` | ❌ | false | 详细日志 |

### LLM 配置（可选）

需要 LLM Judge 时，创建配置文件：

```yaml
# llm_config.yaml
llm:
  default: deepseek_judge
  providers:
    deepseek_judge:
      provider: deepseek
      model: deepseek-chat
      api_key: ${DEEPSEEK_API_KEY}      # 支持 ${ENV_VAR} 语法
      base_url: https://api.deepseek.com/v1
```

不提供 LLM 配置时，LLM 评估器自动降级（score = 0.7，PASS）。

## 四、输出结构

```
workspace/
├── runs/{run_id}/
│   ├── results/{task_id}/
│   │   ├── manifest.json       # 评估结果元信息
│   │   ├── rule_results.json   # 14 项评估器逐项结果
│   │   ├── scores.json         # S_format, S_common, S_soft, S_pref, Reward
│   │   ├── report.md           # 人类可读报告
│   │   └── evidence/           # LLM Judge 溯源记录
│   └── reports/
│       ├── summary.md          # 聚合报告（DR/CPR/Reward 指标表）
│       └── summary.json
├── cache/evaluation_cache.json # 跨运行缓存（重复评估自动跳过）
└── index/runs_index.json       # 运行索引
```

## 六、验证清单

| # | 检查项 | 验证方法 | 预期结果 |
|---|--------|---------|---------|
| 1 | `report.md` 人类可读 | 查看任务报告 | 含 ✅/❌ 状态、得分表格、约束详情 |
| 2 | `summary.md` 含指标 | 查看聚合报告 | DR/CPR/Reward 指标表 + 阈值对比 |
| 3 | `scores.json` 含 reward | `cat results/*/scores.json` | reward > 0（合格），-3.0（格式失败） |
| 4 | LLM 降级模式 | 不传 `--llm-config` | 质量评估器 reason 含 "降级"，score=0.7 |
| 5 | 缓存生效 | 重复运行同一包 | 第二次秒级完成 |
| 6 | 格式门控短路 | 传入空目录的包 | DR=0.0，后续阶段 SKIP |
| 7 | `--validate` 打包验证 | `pack --validate` | 输出验证结果 |

## 七、14 项评估器一览

| ID | 名称 | 层级 | 方法 | 阶段 |
|----|------|------|------|------|
| `format.response_format` | 文件格式检查 | HARD_GATE | RULE | format |
| `format.document_count` | 文档数量检查 | HARD_GATE | RULE | format |
| `format.structure_compliance` | 结构规范性检查 | HARD_GATE | RULE | format |
| `format.html_validity` | HTML 有效性检查 | HARD_GATE | RULE | format |
| `commonsense.info_accuracy` | 知识准确性检查 | HARD_SCORE | FACT_VERIFY | commonsense |
| `commonsense.chronological_order` | 时序正确性检查 | HARD_SCORE | RULE | commonsense |
| `commonsense.logical_consistency` | 逻辑一致性检查 | HARD_SCORE | LLM_CONSISTENCY | commonsense |
| `commonsense.math_formula` | 公式正确性检查 | HARD_SCORE | MATH_VERIFY | commonsense |
| `commonsense.unit_consistency` | 单位一致性检查 | HARD_SCORE | RULE | commonsense |
| `soft.teaching_logic` | 教学逻辑 | SOFT | LLM_JUDGE | quality |
| `soft.content_diversity` | 内容多样性 | SOFT | LLM_JUDGE | quality |
| `pref.style_preference` | 风格偏好 | PREFERENCE | LLM_JUDGE | quality |
| `pref.depth_preference` | 深度偏好 | PREFERENCE | LLM_JUDGE | quality |
| `pref.request_fulfillment` | 需求满足度 | PREFERENCE | LLM_JUDGE | quality |

**评分公式**：`Reward = S_format + S_common + w3 × S_soft + w4 × S_pref`

- `S_format`：全通过 +1，任一失败 -3
- `S_common`：全通过 +1，任一失败 0
- `S_soft`：加权平均 [0, 1]
- `S_pref`：加权平均 [0, 1]
- 默认 `w3 = w4 = 1.0`
