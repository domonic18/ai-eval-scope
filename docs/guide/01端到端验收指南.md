## 一、配置文件模型

### 1. RuleSet（规则集）— 必需

```yaml
# assets/rules/default_rule_set.yaml
version: "1.0"                          # 必需：版本号
description: "课件生成评估规则集"

dimensions:                              # 可选：评估维度
  - {id: functional, name: "功能性", weight: 1.0}
  - {id: effectiveness, name: "效果性", weight: 1.0}

cascade:                                 # 可选：级联阶段
  - {stage: format_gate, name: "格式门控", stop_on_fail: true}
  - {stage: commonsense_gate, name: "常识门控", stop_on_fail: false}

rules:                                   # 必需：规则列表
  - id: FMT_001
    name: "输出格式有效"
    dimension: functional
    stage: format_gate
    evaluator: format.response_format    # 对应注册的评估器 ID
    params: {allowed_formats: ["md", "html"]}
    weight: 1.0
    penalty_on_fail: -3
  - id: FMT_002
    name: "文档数量合规"
    dimension: functional
    stage: format_gate
    evaluator: format.document_count
    params: {min: 1, max: 20}
    weight: 1.0
```

**RuleSet 字段说明：**

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `version` | `str` | ✅ | 规则集版本号，参与缓存键计算 |
| `description` | `str` | ❌ | 描述文本 |
| `dimensions` | `list[Dimension]` | ❌ | 评估维度列表 |
| `cascade` | `list[CascadeStage]` | ❌ | 级联阶段定义 |
| `rules` | `list[Rule]` | ✅ | 规则列表 |

**Dimension 字段：**

| 字段 | 类型 | 必需 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | `str` | ✅ | — | 维度 ID，如 `functional` |
| `name` | `str` | ✅ | — | 显示名称，如 `功能性` |
| `weight` | `float` | ❌ | `1.0` | 权重，≥0 |

**CascadeStage 字段：**

| 字段 | 类型 | 必需 | 默认值 | 说明 |
|------|------|------|--------|------|
| `stage` | `str` | ✅ | — | 阶段 ID，如 `format_gate` |
| `name` | `str` | ❌ | `""` | 显示名称 |
| `stop_on_fail` | `bool` | ❌ | `false` | 是否在失败时中止级联 |

**Rule 字段：**

| 字段 | 类型 | 必需 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | `str` | ✅ | — | 规则 ID，如 `FMT_001` |
| `name` | `str` | ✅ | — | 显示名称 |
| `dimension` | `str` | ✅ | — | 所属维度 ID |
| `stage` | `str` | ✅ | — | 所属级联阶段 ID |
| `description` | `str` | ❌ | `""` | 规则描述 |
| `evaluator` | `str` | ✅ | — | 评估器 ID，如 `format.response_format` |
| `params` | `dict` | ❌ | `{}` | 评估器参数 |
| `weight` | `float` | ❌ | `1.0` | 权重，≥0 |
| `penalty_on_fail` | `float\|None` | ❌ | `None` | 失败惩罚分，如 `-3` |

> **关键说明**：当前 `build_default_pipeline()` 使用**硬编码的 17 项评估器**，不读取 RuleSet 中的 rules。RuleSet 主要用于传递 `version`（缓存键）和未来扩展。验收时 `rule_set.yaml` 只需提供 `version` 字段即可。

### 2. LLM Config（LLM 配置）— 可选

```yaml
# llm_config.yaml（从 assets/configs/llm_config.example.yaml 复制）
llm:
  default: deepseek_judge                # 必需：默认 provider 名称
  providers:                             # 必需：provider 映射
    deepseek_judge:
      provider: deepseek                 # 协议：deepseek | openai | anthropic
      model: deepseek-chat              # 必需：模型 ID
      api_key: ${DEEPSEEK_API_KEY}      # 必需：支持 ${ENV_VAR} 语法
      base_url: https://api.deepseek.com/v1
      max_tokens: 4096
      temperature: 0.0
      seed: 42
```

**LLMConfig 字段：**

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `default` | `str` | ✅ | 默认 provider 名称（必须是 `providers` 中的 key） |
| `providers` | `dict[str, ProviderConfig]` | ✅ | provider 配置映射 |

**ProviderConfig 字段：**

| 字段 | 类型 | 必需 | 默认值 | 说明 |
|------|------|------|--------|------|
| `provider` | `str` | ✅ | — | 协议类型：`deepseek` / `openai` / `anthropic` |
| `model` | `str` | ✅ | — | 模型 ID |
| `api_key` | `str` | ✅ | — | API Key，支持 `${ENV_VAR}` 语法 |
| `base_url` | `str\|None` | ❌ | `None` | API 基础 URL |
| `max_tokens` | `int` | ❌ | `4096` | 最大输出 token 数 |
| `temperature` | `float` | ❌ | `0.0` | 温度，范围 [0, 2] |
| `seed` | `int` | ❌ | `42` | 随机种子 |
| `extra_params` | `dict` | ❌ | `{}` | 额外参数 |

> **不提供 LLM 配置时**：所有 LLM Judge 评估器自动降级为 Rule-based 模式，评分 0.7。验收时可先不带 LLM 运行。

## 二、准备 ExecutionPackage（手动打包）

### 方式 A：Python SDK 打包

```python
from pathlib import Path
from agent_eval.execution.models import Task
from agent_eval.storage.builder import PackageBuilder

# 定义任务
task = Task(
    id="math_001",
    input={"subject": "math", "grade": 7, "topic": "一元一次方程"},
    constraints={"min_documents": 1, "max_documents": 10},
)

# 方式 1：内联模式（指定具体文件列表）
pkg_dir = PackageBuilder.build_inline(
    task=task,
    output_files=[
        "/path/to/your/output/index.md",
        "/path/to/your/output/chapter_01.md",
    ],
    package_dir=Path("workspace/packages/math_001"),
)

# 方式 2：目录模式（自动遍历整个目录）
pkg_dir = PackageBuilder.build_directory(
    task=task,
    source_dir=Path("/path/to/your/output/"),
    package_dir=Path("workspace/packages/math_001"),
)
```

**两种模式对比：**

| 模式 | 适用场景 | 产出差异 |
|------|----------|----------|
| `build_inline` | 已知具体文件列表 | 标准包结构 |
| `build_directory` | 目录结构化产出物 | 额外生成 `output/_manifest.json` 清单 |

### 方式 B：手动创建目录结构

```
workspace/packages/your_task/
├── manifest.json        # 包元信息
├── task.json            # 任务定义
├── metadata.json        # 环境信息
├── output/              # Agent 产出物（MD/HTML 文件）
│   ├── index.md
│   └── chapter_01.md
```

**manifest.json**：

```json
{
  "package_id": "pkg_test_001",
  "created_at": "2026-06-09T12:00:00+00:00",
  "task_id": "math_001",
  "sut_config_id": "manual",
  "status": "success"
}
```

**task.json**：

```json
{
  "id": "math_001",
  "input": {"subject": "math", "grade": 7, "topic": "一元一次方程"},
  "constraints": {}
}
```

**metadata.json**：

```json
{
  "sut_name": "manual",
  "eval_system_version": "0.1.0"
}
```

## 三、运行评估

### CLI 方式

```bash
# 最简模式（无 LLM，评估器降级）
uv run agent-eval eval \
  --package-dir ./workspace/packages/your_task \
  --rule-set ./assets/rules/default_rule_set.yaml

# 完整模式（含 LLM Judge）
uv run agent-eval eval \
  --package-dir ./workspace/packages/your_task \
  --rule-set ./assets/rules/default_rule_set.yaml \
  --llm-config ./llm_config.yaml \
  --llm-provider deepseek_judge \
  --output-dir ./workspace \
  --project "math_courseware" \
  --verbose

# agent 模式（未实现，应提示"尚未实现"）
uv run agent-eval eval \
  --package-dir ./test \
  --rule-set ./assets/rules/default_rule_set.yaml \
  --eval-mode agent
```

**CLI 参数说明：**

| 参数 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `--package-dir` | ✅ | — | ExecutionPackage 目录路径 |
| `--rule-set` | ✅ | — | 规则集 YAML 文件路径 |
| `--output-dir` | ❌ | `./workspace` | 输出目录 |
| `--eval-mode` | ❌ | `pipeline` | 评估模式：`pipeline`（默认）/ `agent`（未实现） |
| `--llm-provider` | ❌ | 默认 provider | 覆盖 LLM Provider 名称 |
| `--llm-config` | ❌ | — | LLM 配置文件路径 |
| `--project` | ❌ | — | 项目 ID（用于 workspace index） |
| `--verbose` / `-v` | ❌ | `false` | 详细日志输出 |

### Python SDK 方式

```python
from agent_eval.orchestrator import eval_packages

# 无 LLM（降级模式）
result = eval_packages(
    package_dir="./workspace/packages/your_task",
    rule_set_path="./assets/rules/default_rule_set.yaml",
    output_dir="./workspace",
    project="math_courseware",

# 含 LLM Judge
result = eval_packages(
    package_dir="./workspace/packages/your_task",
    rule_set_path="./assets/rules/default_rule_set.yaml",
    llm_config_path="./llm_config.yaml",
    llm_provider="deepseek_judge",

# 查看结果
print(f"DR: {result.report.dr}")
print(f"CPR: {result.report.cpr}")
print(f"Avg Reward: {result.report.avg_reward}")
print(f"Results saved to: {result.run_workspace.root}")
```

## 四、输出结构

运行后检查 workspace 目录：

```
workspace/
├── runs/{run_id}/
│   ├── run_manifest.json
│   ├── results/{task_id}/
│   │   ├── manifest.json       # 评估结果元信息
│   │   ├── rule_results.json   # 每个约束的通过/失败详情
│   │   ├── scores.json         # S_format, S_common, S_soft, S_pref, Reward
│   │   ├── report.md           # 人类可读 Markdown 报告
│   │   ├── report.json         # 机器可解析 JSON 报告
│   │   └── evidence/           # LLM Judge 溯源记录（如有）
│   └── reports/
│       ├── summary.md          # 聚合 Markdown 报告
│       └── summary.json        # 聚合 JSON 报告
├── cache/
│   └── evaluation_cache.json   # 跨运行缓存
└── index/
    └── runs_index.json         # Web Portal 索引
```

**各文件用途：**

| 文件 | 格式 | 用途 |
|------|------|------|
| `results/{id}/manifest.json` | JSON | 评估结果元信息（result_id, package_id, rule_set_version, evaluated_at） |
| `results/{id}/rule_results.json` | JSON | 每个约束的详细结果（constraint_id, passed, score, reason, judge_*） |
| `results/{id}/scores.json` | JSON | 评分快照（s_format, s_common, s_soft, s_pref, reward） |
| `results/{id}/report.md` | Markdown | 人类可读报告，含约束状态表格和 LLM 溯源 |
| `results/{id}/report.json` | JSON | 机器可解析的完整 SampleResult |
| `reports/summary.md` | Markdown | 聚合报告，含 DR/CPR/Reward 指标表和失败明细 |
| `reports/summary.json` | JSON | 机器可解析的 MetricsReport |
| `cache/evaluation_cache.json` | JSON | 跨运行的样本级缓存（SHA256 键） |
| `index/runs_index.json` | JSON | 所有运行索引（供 Web Portal 读取） |

## 五、验证清单

| # | 检查项 | 验证方法 | 预期结果 |
|---|--------|---------|---------|
| 1 | `rule_results.json` 可解析 | `python -m json.tool results/*/rule_results.json` | JSON 格式正确，含 `constraint_id`, `passed`, `score` |
| 2 | `scores.json` 含 reward | 检查 `reward` 字段 | 合法文档 reward > 0，格式失败 reward = -3.0 |
| 3 | `report.md` 人类可读 | 打开查看 | 含 ✅/❌ 状态图标、得分表格、约束详情 |
| 4 | `summary.json` 含 DR/CPR | 检查 `metrics.DR` / `metrics.CPR` | 值在 [0, 1] 范围内 |
| 5 | 缓存生效 | 重复运行同一包 | 第二次运行秒级完成，无重复评估 |
| 6 | `--eval-mode agent` | 传入 `--eval-mode agent` | 提示 "Agent 评估模式尚未实现"，exit code 1 |
| 7 | LLM 降级模式 | 不传 `--llm-config` | LLM 评估器 reason 含 "降级"，score = 0.7 |
| 8 | 格式门控短路 | 传入无 output 的包 | DR = 0.0，commonsense/quality 阶段 SKIP |
| 9 | workspace index | 检查 `index/runs_index.json` | 含运行记录、metrics、project |
| 10 | EvaluationResult 可加载 | `EvaluationResult.load(result_dir)` | 成功加载，字段完整 |

## 七、已注册的 17 项评估器

| ID | 名称 | 层级 | 方法 | 阶段 |
|----|------|------|------|------|
| `format.response_format` | 文件格式检查 | HARD_GATE | RULE | format |
| `format.document_count` | 文档数量检查 | HARD_GATE | RULE | format |
| `format.structure_compliance` | 结构规范性检查 | HARD_GATE | RULE | format |
| `format.html_validity` | HTML 有效性检查 | HARD_GATE | RULE | format |
| `format.directory_structure` | 目录结构检查 | HARD_GATE | RULE | format |
| `commonsense.info_accuracy` | 知识准确性检查 | HARD_SCORE | FACT_VERIFY | commonsense |
| `commonsense.chronological_order` | 时序正确性检查 | HARD_SCORE | RULE | commonsense |
| `commonsense.logical_consistency` | 逻辑一致性检查 | HARD_SCORE | LLM_CONSISTENCY | commonsense |
| `commonsense.math_formula` | 公式正确性检查 | HARD_SCORE | MATH_VERIFY | commonsense |
| `commonsense.unit_consistency` | 单位一致性检查 | HARD_SCORE | RULE | commonsense |
| `soft.content_density` | 内容密度 | SOFT | RULE | quality |
| `soft.visual_consistency` | 视觉一致性 | SOFT | RULE | quality |
| `soft.teaching_logic` | 教学逻辑 | SOFT | LLM_JUDGE | quality |
| `soft.content_diversity` | 内容多样性 | SOFT | LLM_JUDGE | quality |
| `pref.style_preference` | 风格偏好 | PREFERENCE | LLM_JUDGE | quality |
| `pref.depth_preference` | 深度偏好 | PREFERENCE | LLM_JUDGE | quality |
| `pref.request_fulfillment` | 需求满足度 | PREFERENCE | LLM_JUDGE | quality |

**评分公式**：`Reward = S_format + S_common + w3 × S_soft + w4 × S_pref`

- `S_format`：全通过 +1，任一失败 -3
- `S_common`：全通过 +1，任一失败 0
- `S_soft`：加权平均 [0, 1]
- `S_pref`：加权平均 [0, 1]
- 默认 `w3 = w4 = 1.0`
