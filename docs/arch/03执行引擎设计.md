# 执行引擎设计

> 本文档详细阐述执行引擎的设计，核心变更为：执行引擎全面采用 **Agent-Driven** 架构，基于 Claude Agent SDK 的 ExecutionAgent 端到端驱动评测执行流程，SUT 交互能力以 MCP Tools 形式暴露。属于 [01 整体架构设计](./01整体架构设计.md) §二中"执行引擎"的详细展开。

---

## 一、职责边界

执行引擎负责**驱动被测 Agent 运行并采集结果**，不涉及任何评估逻辑：

| 组件 | 职责 |
|------|------|
| **ExecutionAgent** | 基于 Claude Agent SDK，端到端驱动执行流程：理解任务、调用 SUT Tools、处理错误、收集结果、生成 ExecutionPackage |
| **SUTToolServer** | 提供 MCP Tools（HTTP 调用、CLI 执行、目录扫描、文件读取、结果收集等），供 ExecutionAgent 调用 |
| **DirectoryCollector** | 遍历目录结构、收集 HTML 文件、生成目录清单（通过 `scan_directory` MCP Tool 暴露给 Agent） |
| **TaskSetBuilder** | 任务集构建工具（从零创建 / LLM 半自动生成） |

> **架构变更说明**：传统设计中的 ExecutionEngine + SUTDriver（HTTPDriver/CLIDriver/AgentSUTDriver）分层架构已被 ExecutionAgent + SUT Tools 的 Agent-Driven 架构取代。HTTPDriver/CLIDriver 的逻辑不再作为独立的 Driver 类存在，而是下沉为 SUTToolServer 中的 `invoke_http_sut` 和 `invoke_cli_sut` MCP Tool。ExecutionAgent 根据任务描述智能选择并调用合适的 Tool。

## 三、ExecutionAgent — Agent 驱动执行

### 3.1 设计理念

传统执行引擎采用 ExecutionEngine + SUTDriver 分层架构：引擎调度 Driver，Driver 机械地渲染模板、发送请求、解析响应。这种方式在简单场景下工作良好，但面对以下挑战时力不从心：

- **任务多样性**：不同 SUT 的接口格式、认证方式、错误模式各不相同，硬编码逻辑难以覆盖
- **错误恢复**：网络超时、服务降级、参数不匹配等问题需要智能判断和处理策略
- **上下文理解**：某些任务需要理解任务描述的语义，动态调整请求构造方式

**Agent-Driven 方案的核心思想**：将执行控制权交给一个理解任务语义的智能 Agent（基于 Claude Agent SDK），由 Agent 根据任务描述自主决定调用哪些 SUT Tools、如何构造请求参数、遇到错误时如何重试或降级，最终生成结构化的 ExecutionPackage。

**与传统方案的对比**：

| 维度 | 传统 ExecutionEngine + SUTDriver | Agent-Driven ExecutionAgent |
|------|----------------------------------|------------------------------|
| 执行控制 | 代码硬编码的固定流程 | Agent 理解任务后自主决策 |
| 请求构造 | Jinja2 模板变量替换 | Agent 理解任务语义后智能构造 |
| 错误处理 | 预定义的重试/超时策略 | Agent 根据错误类型动态决策 |
| 结果收集 | 固定的文件复制/解析逻辑 | Agent 调用 scan_directory / collect_results 灵活收集 |
| 适应性 | 新 SUT 需编写新 Driver | 新 SUT 只需配置 Tool 参数 |
| 可观测性 | 日志记录 | 完整的 Agent 会话记录（所有决策过程可见） |

### 3.2 核心类

```python
from claude_agent_sdk import query, ClaudeAgentOptions


class ExecutionAgent:
    """基于 Claude Agent SDK 的执行 Agent，端到端驱动评测执行流程。

    ExecutionAgent 接收 Task，通过调用 SUT Tools（MCP）与被测系统交互，
    自主完成请求构造、错误处理、结果收集，最终生成 ExecutionPackage。
    """

    def __init__(self, config: AgentConfig, sut_tools: SUTToolServer):
        self.config = config
        self.sut_tools = sut_tools

    async def run_task(self, task: Task) -> ExecutionPackage:
        """执行单个任务，返回 ExecutionPackage。"""
        system_prompt = self._build_system_prompt(task)
        options = ClaudeAgentOptions(
            system_prompt=system_prompt,
            mcp_servers={"sut-tools": self.sut_tools.to_mcp_server()},
            allowed_tools=self.sut_tools.get_tool_names(),
            max_turns=self.config.max_turns,
            max_budget_usd=self.config.budget_usd,
            permission_mode="acceptEdits",
        )
        prompt = self._build_task_prompt(task)
        session = await self._run_agent(prompt, options)
        return self._build_package(session, task)

    async def run_task_set(self, task_set: TaskSet) -> list[ExecutionPackage]:
        """批量执行任务集，返回执行包列表。"""
        return [await self.run_task(task) for task in task_set.tasks]

        """运行 Agent 会话，返回会话记录。"""
```

### 3.4 Task Prompt 构建

Task Prompt 提供具体的任务信息，供 Agent 理解并执行：

```python
def _build_task_prompt(self, task: Task) -> str:
    prompt_parts = [
        f"## 任务 ID: {task.id}",
        f"## 任务输入",
        json.dumps(task.input, ensure_ascii=False, indent=2),
    ]
    if task.expected:
        prompt_parts.append(f"## 预期结果\n{json.dumps(task.expected, ensure_ascii=False, indent=2)}")
    if task.constraints:
        prompt_parts.append(f"## 约束条件\n{json.dumps(task.constraints, ensure_ascii=False, indent=2)}")

    # 目录模式
    if task.input_mode == "directory" and task.directory_path:
        prompt_parts.extend([
            f"## 目录模式",
            f"- 目录路径: {task.directory_path}",
            f"- 文件匹配模式: {task.file_patterns}",
            f"请先调用 scan_directory 扫描目录结构，然后收集结果文件。",
        ])

    prompt_parts.append("请执行上述任务，并在完成后调用 write_package 写入执行包。")
    return "\n\n".join(prompt_parts)
```

### 4.2 invoke_http_sut — HTTP 请求工具

继承原 HTTPDriver 的核心能力：Jinja2 模板渲染、请求发送、响应映射。Agent 传入任务参数，工具负责渲染模板并发送请求。

```python
@mcp_tool(name="invoke_http_sut", description="发送 HTTP 请求到 SUT（支持 Jinja2 模板渲染）")
async def invoke_http_sut(
    method: str,                          # HTTP 方法（GET/POST/PUT）
    url: str,                             # 请求 URL
    headers: dict = None,                 # 自定义请求头
    body_template: str = None,            # 请求体模板（Jinja2 格式）
    template_vars: dict = None,           # 模板变量（用于渲染 body_template）
    response_mapping: dict = None,        # 响应字段映射规则
    timeout: float = 120,                 # 超时时间（秒）
) -> dict:
    """
    发送 HTTP 请求到被测 Agent 服务。

    1. 如果提供 body_template + template_vars，使用 Jinja2 渲染请求体
    2. 发送 HTTP 请求（带超时控制）
    3. 如果提供 response_mapping，按映射规则提取输出字段
    4. 返回结构化响应

    示例：
      body_template = '{"subject": "{{ subject }}", "grade": "{{ grade }}"}'
      template_vars = {"subject": "数学", "grade": "七年级"}
      → 渲染结果: '{"subject": "数学", "grade": "七年级"}'
    """
    from jinja2 import Template

    # 渲染请求体
    if body_template and template_vars:
        template = Template(body_template)
        body = template.render(**template_vars)
    else:
        body = body_template

    # 发送请求
    async with httpx.AsyncClient(timeout=timeout) as client:
        response = await client.request(
            method=method, url=url, headers=headers,
            content=body,
        )

    # 映射响应字段
    result = {"status_code": response.status_code, "raw": response.text}
    if response_mapping:
        parsed = response.json()
        for target_field, source_path in response_mapping.items():
            result[target_field] = _extract_by_path(parsed, source_path)

    return result
```

**配置迁移说明**：

| 原 HTTPDriver 配置 | invoke_http_sut 对应参数 |
|---------------------|--------------------------|
| `base_url` | `url` 参数 |
| `headers` | `headers` 参数 |
| `request_template.body` | `body_template` + `template_vars` |
| `response_mapping` | `response_mapping` |
| `timeout` | `timeout` |
| `retries` | 由 ExecutionAgent 自行决定重试策略 |

### 4.3 invoke_cli_sut — CLI 执行工具

继承原 CLIDriver 的能力，通过子进程调用本地 CLI Agent。

```python
@mcp_tool(name="invoke_cli_sut", description="执行 CLI 命令调用本地 SUT")
async def invoke_cli_sut(
    command: str,                         # 完整的 CLI 命令
    working_dir: str = None,              # 工作目录
    env_vars: dict = None,                # 环境变量
    timeout: float = 120,                 # 超时时间（秒）
) -> dict:
    """
    通过子进程执行 CLI 命令调用被测 Agent。

    返回:
      {"exit_code": int, "stdout": str, "stderr": str}
    """
    import asyncio

    proc = await asyncio.create_subprocess_shell(
        command,
        stdout=asyncio.subprocess.PIPE,
        stderr=asyncio.subprocess.PIPE,
        cwd=working_dir,
        env={**os.environ, **(env_vars or {})},
    )
    try:
        stdout, stderr = await asyncio.wait_for(
            proc.communicate(), timeout=timeout
        )
        return {
            "exit_code": proc.returncode,
            "stdout": stdout.decode(),
            "stderr": stderr.decode(),
        }
    except asyncio.TimeoutError:
        proc.kill()
        return {"exit_code": -1, "stdout": "", "stderr": f"命令执行超时（{timeout}秒）"}
```

### 4.5 collect_results — 结果收集工具

```python
@mcp_tool(name="collect_results", description="收集执行结果文件到 workspace")
async def collect_results(
    source_paths: list[str],              # 源文件/目录路径列表
    workspace_dir: str,                   # 目标 workspace 目录
    task_id: str,                         # 任务 ID（用于组织输出目录）
) -> dict:
    """
    将源文件/目录复制到 ExecutionPackage 的 output 目录。

    目录结构：
      {workspace_dir}/{task_id}/output/    ← 输出文件
      {workspace_dir}/{task_id}/_manifest.json  ← 目录清单（如适用）
    """
    output_dir = Path(workspace_dir) / task_id / "output"
    output_dir.mkdir(parents=True, exist_ok=True)

    collected_files = []
    for source in source_paths:
        src = Path(source)
        if src.is_file():
            shutil.copy2(src, output_dir / src.name)
            collected_files.append(str(output_dir / src.name))
        elif src.is_dir():
            shutil.copytree(src, output_dir / src.name, dirs_exist_ok=True)
            collected_files.append(str(output_dir / src.name))

    return {"collected_files": collected_files, "output_dir": str(output_dir)}
```

### 4.6 write_package — 执行包写入工具

```python
@mcp_tool(name="write_package", description="写入 ExecutionPackage 到 workspace")
async def write_package(
    workspace_dir: str,
    task_id: str,
    success: bool,
    output_files: list[str] = None,
    output_directory: str = None,
    metadata: dict = None,
    error: str = None,
    trace: dict = None,
    metrics: dict = None,
) -> dict:
    """
    将 ExecutionPackage 写入 workspace。

    写入文件：
      {workspace_dir}/{task_id}/metadata.json
      {workspace_dir}/{task_id}/trace.json
      {workspace_dir}/{task_id}/metrics.json
    """
    package_dir = Path(workspace_dir) / task_id
    package_dir.mkdir(parents=True, exist_ok=True)

    if metadata:
        (package_dir / "metadata.json").write_text(json.dumps(metadata, ensure_ascii=False, indent=2))
    if trace:
        (package_dir / "trace.json").write_text(json.dumps(trace, ensure_ascii=False, indent=2))
    if metrics:
        (package_dir / "metrics.json").write_text(json.dumps(metrics, ensure_ascii=False, indent=2))

    return {"package_dir": str(package_dir), "success": success}
```

```python
@mcp_tool(name="list_files", description="列出目录中的文件")
async def list_files(
    directory_path: str,
    pattern: str = "*",
    recursive: bool = False,
) -> list[str]:
    """列出目录中匹配指定模式的文件。"""
    base = Path(directory_path)
    if recursive:
        return [str(p.relative_to(base)) for p in base.rglob(pattern) if p.is_file()]
    return [str(p.name) for p in base.glob(pattern) if p.is_file()]
```

## 五、执行流程对比

### 5.1 传统模式 vs Agent-Driven 模式

```
传统模式（ExecutionEngine + SUTDriver）：
  TaskSet → ExecutionEngine.run()
              ├── Task → HTTPDriver.invoke()
              │           ├── Jinja2 渲染模板
              │           ├── HTTP 请求发送
              │           └── 响应解析 → SUTResponse
              ├── Task → CLIDriver.invoke()
              │           ├── 构建命令行参数
              │           └── 子进程执行 → SUTResponse
              ├── ResultCollector.collect()
              └── ExecutionPackage 归档

Agent-Driven 模式（ExecutionAgent + SUT Tools）：
  TaskSet → ExecutionAgent.run_task_set()
              └── Task → ExecutionAgent.run_task()
                          ├── 构建 System Prompt + Task Prompt
                          ├── 启动 Claude Agent 会话
                          ├── Agent 理解任务，自主决策：
                          │   ├── 调用 invoke_http_sut（含 Jinja2 渲染）
                          │   ├── 或调用 invoke_cli_sut
                          │   ├── 或调用 scan_directory → collect_results
                          │   ├── 遇到错误 → 智能重试 / 降级
                          │   └── 调用 write_package 写入结果
                          └── 从 Agent 会话构建 ExecutionPackage
```

### 5.2 流程对比表

| 阶段 | 传统模式 | Agent-Driven 模式 |
|------|----------|-------------------|
| 任务理解 | 代码逻辑解析 Task 字段 | Agent 理解 Task Prompt 语义 |
| 请求构造 | Jinja2 模板变量替换 | Agent 调用 invoke_http_sut，传入模板和变量 |
| SUT 调用 | HTTPDriver/CLIDriver 固定流程 | Agent 选择并调用合适的 MCP Tool |
| 错误处理 | 预定义的重试次数和超时 | Agent 根据错误类型动态决策（重试/降级/终止） |
| 结果收集 | ResultCollector 固定逻辑 | Agent 调用 collect_results / scan_directory |
| 包归档 | ExecutionEngine 内部方法 | Agent 调用 write_package |
| 可观测性 | 日志 + ExecutionTrace | 完整 Agent 会话记录（所有决策和工具调用） |
| 扩展性 | 编写新 SUTDriver 类 | 添加新 MCP Tool 或调整 Prompt |

## 七、异常处理

执行引擎定义以下异常类型，覆盖 Agent-Driven 执行的各类错误场景：

```python
class ExecutionError(Exception):
    """执行引擎基础异常"""
    pass

class AgentError(ExecutionError):
    """ExecutionAgent 执行失败

    适用场景：
    - Claude Agent SDK 调用失败（连接错误、认证失败等）
    - Agent 执行超过 max_turns 限制
    - Agent 产出物解析失败
    - Agent 工具调用权限不足
    """
    pass

class AgentTimeoutError(AgentError):
    """Agent 执行超时（超过 max_turns 或 wall-clock timeout）"""
    pass

class BudgetExceededError(AgentError):
    """Agent 执行超出预算（超过 max_budget_usd）"""
    pass

class ToolExecutionError(ExecutionError):
    """MCP Tool 执行失败（HTTP 请求失败、CLI 执行错误、文件操作失败等）"""
    pass

class CollectionError(ExecutionError):
    """结果采集失败（文件复制错误、目录遍历失败等）"""
    pass

class TaskBuildError(ExecutionError):
    """任务集构建失败（模板渲染错误、LLM 生成异常等）"""
    pass
```

**Agent 错误处理策略**：

| 错误类型 | 处理策略 |
|----------|----------|
| Tool 调用失败（HTTP 超时/CLI 错误） | Agent 自主重试（最多 `max_retries` 次），或降级到其他工具 |
| 超过 max_turns | 抛出 `AgentTimeoutError`，写入包含部分结果的 ExecutionPackage |
| 超过 max_budget_usd | 抛出 `BudgetExceededError`，立即终止执行 |
| Agent 会话异常中断 | 捕获异常，记录已完成的工具调用轨迹，写入错误 ExecutionPackage |
| 工具返回意外结果 | Agent 分析错误信息，尝试修正参数后重试 |
