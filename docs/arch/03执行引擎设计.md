# 执行引擎设计

> 本文档详细阐述执行引擎的设计，包括 SUT 驱动、结果采集、过程指标统计以及执行侧的数据模型。属于 [01 整体架构设计](./01整体架构设计.md) §二中"执行引擎"的详细展开。

---

## 一、职责边界

执行引擎负责**驱动被测 Agent 运行并采集结果**，不涉及任何评估逻辑：

| 组件 | 职责 |
|------|------|
| **ExecutionEngine** | 加载任务集、调度 SUTDriver、归档输出、统计过程指标 |
| **SUTDriver** | 与被测 Agent 通信（HTTP / CLI / 扩展） |
| **ResultCollector** | 收集输出文件、记录执行轨迹 |
| **DirectoryCollector** | 遍历目录结构、收集 HTML 文件、生成目录清单 |
| **MetricRecorder** | 统计耗时、步数、重试次数等过程指标 |
| **TaskSetBuilder** | 任务集构建工具（从零创建 / LLM 半自动生成） |

### 2.2 Task — 单个任务

```python
class Task(BaseModel):
    id: str                              # 任务唯一标识
    input: dict                          # 输入参数（学科、年级、知识点等）
    expected: dict | None = None         # 预期结果（知识点清单、约束等）
    constraints: dict = {}               # 约束条件（文档数量、格式等）
    # 目录模式字段（当评估目标为目录结构时使用）
    input_mode: str = "inline"           # "inline"（默认）| "directory"（目录模式）
    directory_path: str | None = None    # 目录路径（input_mode="directory" 时必填）
    file_patterns: list[str] = ["*.html"] # 文件匹配模式（如 ["*.html", "*.htm"]）
```

class SUTResponse(BaseModel):

## 三、ExecutionEngine

### 3.1 核心类

```python
class ExecutionEngine:
    def __init__(self, driver: SUTDriver, output_dir: Path):
        self.driver = driver
        self.output_dir = output_dir

    def run(self, task_set: TaskSet) -> list[ExecutionPackage]:
        """批量执行任务集，返回执行包列表。"""
        return [self.run_single(task) for task in task_set.tasks]

    def run_single(self, task: Task) -> ExecutionPackage:
        """执行单个任务，归档输出到 workspace。"""
        trace = ExecutionTrace(started_at=datetime.now().isoformat())
        try:
            response = self.driver.invoke(task)
            trace.finished_at = datetime.now().isoformat()
            trace.response = response.raw_response
            trace.error = response.error
            ...
        except Exception as e:
            trace.error = str(e)
            ...
```

## 四、SUTDriver — 被测 Agent 驱动

### 4.1 抽象接口

```python
class SUTDriver(ABC):
    @abstractmethod
    def invoke(self, task: Task) -> SUTResponse: ...
```

### 4.2 HTTPDriver（默认）

适配 LangGraph / Hermes 等 HTTP 服务。

```python
class HTTPDriver(SUTDriver):
    def __init__(self, base_url: str, headers: dict = None,
                 request_template: dict = None, response_mapping: dict = None,
                 timeout: float = 120, retries: int = 2):
        self.base_url = base_url
        self.headers = headers or {}
        self.request_template = request_template or {}
        self.response_mapping = response_mapping or {}
        self.timeout = timeout
        self.retries = retries

    def invoke(self, task: Task) -> SUTResponse:
        # 1. 渲染请求模板（Jinja2 变量替换）
        body = self._render_template(self.request_template.get("body", {}), task)
        # 2. 发送 HTTP 请求（带超时与重试）
        response = self._send_request(body)
        # 3. 提取输出文件与元数据
        return self._parse_response(response)
```

**配置项**：

| 配置 | 说明 | 示例 |
|------|------|------|
| `base_url` | 被测 Agent 服务地址 | `http://localhost:8000` |
| `headers` | 自定义请求头（鉴权等） | `Authorization: Bearer xxx` |
| `request_template` | 请求体模板（Jinja2） | `{subject: "{task.input.subject}"}` |
| `response_mapping` | 响应字段映射 | `output_files_field: "files"` |
| `timeout` | 超时时间（秒） | `120` |
| `retries` | 重试次数 | `2` |

> **注意**：当前阶段 Agent 产出物（Markdown/HTML 文档集）以手动放置为主，HTTPDriver 作为后续自动化执行的预留能力。详见 [迭代开发计划](../plan/01迭代开发计划.md) Sprint 8。

### 4.4 DirectoryCollector — 目录结构采集

当评估目标为包含目录结构的 HTML 文件集时（如课件生成的"大单元学习总导"目录），使用 DirectoryCollector 自动遍历并收集文件。

```python
@dataclass
class CollectedFile:
    """采集到的单个文件信息"""
    relative_path: str              # 相对于根目录的路径
    absolute_path: Path
    file_size: int
    depth: int                      # 目录深度（根目录为 0）
    parent_module: str              # 直接父目录名
    file_type: str                  # 扩展名（如 ".html"）

@dataclass
class DirectoryManifest:
    """目录清单 — 记录完整的目录层级与文件信息"""
    root_dir: str                   # 根目录路径
    total_files: int                # 文件总数
    file_types: dict[str, int]      # 按类型统计（如 {".html": 24}）
    modules: list[ModuleInfo]       # 顶层模块列表
    files: list[CollectedFile]      # 所有采集到的文件（扁平列表）
    hierarchy: dict                 # 嵌套字典表示的目录树
    hierarchy_depth: int            # 最大目录深度

@dataclass
class ModuleInfo:
    """顶层模块信息"""
    name: str                       # 模块名（如 "M1 大单元概述"）
    path: str                       # 相对路径
    file_count: int                 # 该模块下的文件数
    children: list[CollectedFile]   # 模块下所有文件

class DirectoryCollector:
    """遍历目录结构，收集指定类型的文件，保留层级信息。"""

    def __init__(self, root_dir: Path, file_patterns: list[str] = None):
        self.root_dir = root_dir
        self.file_patterns = file_patterns or ["*.html", "*.htm"]

    def collect(self) -> DirectoryManifest:
        """遍历目录树，收集所有匹配文件，返回目录清单。"""
        ...

    def collect_flat(self) -> list[CollectedFile]:
        """返回扁平化的文件列表（不含层级信息）。"""
        ...
```

**使用场景**：

```
输入：/path/to/大单元学习总导/     ← Task.directory_path
     ├── M1 大单元概述/
     │   └── 大单元概述/
     │       ├── 整体定位/整体定位.html
     │       └── ...
     └── M2 学习新知的新价值与新使命/
         ├── 建构性导学/...
         └── ...

输出：DirectoryManifest
     ├── modules: [ModuleInfo("M1 大单元概述", 6文件), ModuleInfo("M2 ...", 18文件)]
     ├── total_files: 24
     └── hierarchy_depth: 3
```

**与 PackageBuilder 的集成**：在当前"评估优先"阶段，PackageBuilder 新增 `--directory` 模式，内部调用 DirectoryCollector 遍历指定目录，将文件复制到 ExecutionPackage 的 `output/` 目录，并生成 `output/_manifest.json`。

> **注意**：当前阶段 DirectoryCollector 主要通过 PackageBuilder 使用。Sprint 8 执行引擎就绪后，LocalDirectoryDriver 将直接集成此组件。
