# 数据管理与配置规范

> 本文档定义系统的数据管理与配置体系，涵盖预置规则数据、运行时数据、配置加载机制三个维度。属于 [01 整体架构设计](./01整体架构设计.md) §二中"基础设施层"的详细展开。评估引擎参见 [04 评估引擎设计](./04评估引擎设计.md)，LLM Judge 参见 [05 LLM 模块设计](./05LLM模块设计.md)。

## 一、总览

### 1.1 数据分类

系统中的数据按生命周期和用途分为三类：

| 分类 | 说明 | 存储位置 |
|------|------|----------|
| **预置规则数据** | 规则集、知识库、Prompt 模板等静态配置资产 | `assets/` |
| **运行数据** | 执行包、评估结果、缓存、索引等运行时产生的数据 | `workspace/` |
| **配置管理** | 系统配置加载、路径管理、Schema 校验等基础设施 | `agent_eval/config/` |

### 1.2 目录布局

```
agent-eval-system/
├── assets/                        # 预置规则数据（入版本控制）
│   ├── knowledge/                 # 学科知识库 YAML
│   │   ├── _defaults.yaml         # 通用默认知识（常数、误解等）
│   │   ├── math.yaml              # 数学学科知识
│   │   ├── physics.yaml           # 物理学科知识
│   │   └── ...                    # 其他学科
│   ├── prompts/                   # LLM Judge Prompt 模板 YAML
│   │   ├── pedagogical_logic.yaml
│   │   ├── content_diversity.yaml
│   │   └── ...
│   ├── rules/                     # 默认规则集
│   │   └── default_rule_set.yaml
│   ├── schemas/                   # JSON Schema 校验文件
│   │   ├── rule_set_schema.json
│   │   ├── task_set_schema.json
│   │   └── llm_config_schema.json
│   └── configs/                   # 配置样例
│       └── llm_config.example.yaml
│
├── workspace/                     # 运行数据（.gitignore 排除）
│   ├── runs/{run_id}/             # 运行目录
│   ├── packages/                  # 已打包的执行包
│   ├── datasets/{name}/           # 评测数据集（agent-eval dataset download 下载）
│   ├── cache/evaluation_cache.json  # 跨运行评估缓存
│   └── index/runs_index.json      # 运行索引
│
└── agent_eval/config/             # 配置管理模块
    ├── loader.py                  # ConfigLoader
    └── paths.py                   # ProjectPaths
```

## 二、预置规则数据

### 2.1 规则集配置（rule_set.yaml）

定义评估规则的具体约束条件，与评估器 ID 一一对应。

```yaml
# assets/rules/default_rule_set.yaml
version: "1.0"
description: "课件生成评估规则集"
schema_ref: "./assets/schemas/rule_set_schema.json"

dimensions:
  - id: functional
    name: "功能性"
    weight: 1.0
  - id: effectiveness
    name: "效果性"
    weight: 1.0

cascade:
  - stage: format
    name: "格式门控"
    stop_on_fail: true
  - stage: commonsense
    name: "常识门控"
    stop_on_fail: false
  - stage: quality
    name: "质量评估"
    stop_on_fail: false

rules:
  - id: FMT_001
    name: "输出格式有效"
    dimension: functional
    stage: format
    evaluator: format.response_format
    params:
      allowed_formats: ["md", "html"]
    weight: 1.0
    penalty_on_fail: -3

  - id: FMT_002
    name: "文档数量合规"
    dimension: functional
    stage: format
    evaluator: format.document_count
    params:
      min: 1
      max: 30
    weight: 1.0
    penalty_on_fail: -3

  # ... 更多规则
```

**数据模型**（`agent_eval/rules/models.py`，Pydantic BaseModel）：

```python
class RuleSet(BaseModel):
    version: str
    description: str = ""
    schema_ref: str = ""
    dimensions: list[Dimension]
    cascade: list[CascadeStage]
    rules: list[Rule]

    def get_rules_by_stage(self, stage_id: str) -> list[Rule]: ...
    def get_cascade_stage(self, stage_id: str) -> CascadeStage | None: ...
    def get_rule(self, rule_id: str) -> Rule | None: ...
```

### 2.2 知识库配置（knowledge/*.yaml）

知识库为常识评估器（特别是 `commonsense.info_accuracy`）提供事实性数据支撑。采用多文件分层架构：

```
assets/knowledge/
├── _defaults.yaml         # 通用默认知识（所有学科共享）
├── math.yaml              # 数学：常数、公式、误解
├── physics.yaml           # 物理：常数、单位换算
├── chemistry.yaml         # 化学
├── biology.yaml           # 生物
├── history.yaml           # 历史
├── geography.yaml         # 地理
├── chinese.yaml           # 语文
└── morality.yaml          # 道德与法治
```

**加载策略**：

- `_defaults.yaml` 始终加载，包含通用常数和误解列表
- 按任务 `subject` 字段按需加载对应学科文件
- 模块级缓存避免重复 IO

**知识库 YAML 结构**：

```yaml
# assets/knowledge/math.yaml
constants:
  - name: "圆周率"
    value: "3.14159265358979..."
    aliases: ["π", "pi"]
  - name: "自然对数的底"
    value: "2.71828..."
    aliases: ["e"]

misconceptions:
  - pattern: "0.999...≠1"
    correction: "0.999... = 1，这是极限的严格定义"
  - pattern: "负数没有平方根"
    correction: "负数在复数范围内有平方根，如 √(-1) = i"

    - name: "勾股定理"

      formula: "x = (-b ± √(b²-4ac)) / 2a"
```

### 2.3 Prompt 模板配置（prompts/*.yaml）

LLM Judge 的评估 Prompt 模板，每个模板对应一个或多个 LLM Judge 评估器。

```yaml
# assets/prompts/pedagogical_logic.yaml
template_id: pedagogical_logic
name: "教学逻辑评估"

dimensions:
  - dim_id: goal_alignment
    name: "教学目标对齐度"
    description: "课件内容是否与声明的教学目标一致"
    weight: 0.25
    score_range: [0, 10]
  # ... 更多维度

system_prompt: |
  你是一位资深教育专家。请对以下课件进行教学适用性评估。

user_prompt_template: |
  请评估以下课件的【教学逻辑】质量：
  【学科】{subject}
  【年级】{grade}
  ...
  {document_content}

output_schema:
  type: object
  properties: ...

temperature: 0.0
seed: 42
num_samples: 3
```

**数据模型**（`agent_eval/llm/judge/template_manager.py`）：

```python
@dataclass
class JudgeDimension:
    dim_id: str
    name: str
    description: str
    weight: float
    score_range: tuple[float, float] = (0, 10)

@dataclass
class JudgeTemplate:
    template_id: str
    name: str
    dimensions: list[JudgeDimension]
    system_prompt: str

    seed: int = 42

```

### 2.4 JSON Schema 校验

系统提供 JSON Schema 用于校验配置文件的合法性：

| Schema 文件 | 校验对象 |
|-------------|----------|
| `assets/schemas/rule_set_schema.json` | rule_set.yaml |
| `assets/schemas/task_set_schema.json` | task_set.yaml（含目录模式字段） |
| `assets/schemas/llm_config_schema.json` | llm_config.yaml |

Schema 校验在配置加载时自动执行，支持 IDE 实时提示和 CLI 启动时前置检查。

### 3.6 Workspace 生命周期

| 阶段 | 说明 |
|------|------|
| **创建** | 每次 `agent-eval` 运行自动在 `workspace/runs/` 下创建以时间戳命名的运行目录 |
| **run-only 模式** | 仅写入 `packages/` |
| **eval-only 模式** | 读取 `packages/`，写入 `results/` + `reports/`（**当前主要使用模式**） |
| **pipeline 模式** | 完整写入 `packages/` → `results/` → `reports/` |
| **复用** | 历史运行包可用于重新评估、人工复评、基线对比 |
| **清理** | Workspace 不入版本控制，可安全删除历史运行 |

### 3.7 缓存机制

`workspace/cache/evaluation_cache.json` 存储跨运行的评估结果缓存，避免重复计算。

**缓存策略**：

| 维度 | 说明 |
|------|------|
| **缓存键** | SHA256（样本内容 + 规则版本 + 约束条件） |
| **加载时机** | `PipelineEngine.evaluate_batch()` 开始前，从磁盘加载到 `PipelineEngine._cache` |
| **保存时机** | 批量评估结束后，`Orchestrator` 持久化到 `evaluation_cache.json` |
| **缓存粒度** | 样本级（每个样本独立缓存） |
| **缓存命中** | 相同样本 + 相同规则版本的重复评估直接返回缓存结果 |

### 3.8 索引机制

`workspace/index/runs_index.json` 维护运行摘要索引，用于 Web Portal 项目看板与趋势追踪。

**数据结构**：

```json
{
  "runs": [
    {
      "run_id": "20260608_143000",
      "project": "courseware-platform",
      "created_at": "2026-06-08T14:30:00Z",
      "total_samples": 50,
      "metrics": {
        "DR": 0.94,
        "CPR": 0.88,
        "avg_reward": 1.72
      },
      "failure_breakdown": {"format.document_count": 3},
      "rule_set_version": "courseware_v1.2"
    }
  ]
}
```

**索引维护**：

| 时机 | 操作 |
|------|------|
| 每次 `agent-eval eval` 运行完成 | `Orchestrator` 自动追加到 `runs_index.json` |
| `agent-eval index` 命令 | 全量扫描 workspace/runs/，重建完整索引 |

## 四、配置管理

### 4.1 配置文件总览

| 配置文件 | 存放位置 | 作用 |
|----------|----------|------|
| `pipeline.yaml` | `assets/rules/` | 管线配置：阶段、评估器、权重、阈值（**被 Markdown 评测计划替代**） |
| **`eval_plan.md`** | `assets/plans/` | Markdown 评测计划：YAML frontmatter + Markdown 正文 |
| `rule_set.yaml` | `assets/rules/` | 规则集：具体规则定义 |
| `task_set.yaml` | `assets/tasks/` | 任务集：输入数据定义 |
| `project.yaml` | `assets/projects/` | 项目配置：评估运行分组 |
| `sut_config.yaml` | CLI 参数或 `assets/` | 被测 Agent 连接配置 |
| `llm_config.yaml` | CLI 参数或 `assets/` | LLM Provider 配置 |

### 4.2 ConfigLoader

`agent_eval/config/loader.py` 提供统一的配置加载 API：

```python
class ConfigLoader:
    """配置加载器 — YAML 加载 + JSON Schema 校验"""

    @staticmethod
    def load_yaml(path: Path) -> dict:
        """通用 YAML 加载。"""

    @staticmethod
    def validate_schema(data: dict, schema_path: Path) -> list[dict]:
        """JSON Schema 校验，返回校验错误列表。"""

    @staticmethod
    def load_and_validate(path: Path, schema_path: Path) -> dict:
        """加载 YAML 并校验 Schema。"""

    @staticmethod
    def load_rule_set(path: Path, schema_path: Path = None) -> RuleSet:
        """加载规则集为 RuleSet Pydantic 模型。"""

    @staticmethod
    def load_task_set(path: Path, schema_path: Path = None) -> TaskSet:
        """加载任务集为 TaskSet Pydantic 模型。"""

    @staticmethod
    def load_llm_config(path: Path, schema_path: Path = None) -> LLMConfig:
        """加载 LLM 配置为 LLMConfig 模型（兼容顶层 llm: 键）。"""

    @staticmethod
    def get_schema_path(schema_name: str) -> Path:
        """获取 Schema 文件路径。"""
```

### 4.4 管线配置（pipeline.yaml）

定义评估管线的阶段、评估器和权重。当前被 `eval_plan.md` 替代，作为兼容回退。

```yaml
pipeline:
  reward_weights:
    w3: 1.0    # 软约束乘数
    w4: 1.0    # 偏好约束乘数

  stages:
    - id: format
      short_circuit_policy: fail_fast
      evaluators:
        - name: format.response_format
        - name: format.document_count
        - name: format.structure_compliance
        - name: format.html_validity

    - id: commonsense
      short_circuit_policy: fail_fast
      evaluators:
        - name: commonsense.info_accuracy
        - name: commonsense.chronological_order
        - name: commonsense.logical_consistency
        - name: commonsense.math_formula
        - name: commonsense.unit_consistency

    - id: quality
      short_circuit_policy: continue_all
      evaluators:
```

### 4.9 Markdown 评测计划（eval_plan.md）

采用 YAML frontmatter + Markdown 正文的混合格式，替代 `pipeline.yaml` 的级联阶段定义：

```markdown
---
plan_id: courseware-eval-v2
version: "2.0.0"
target: courseware-generation
max_turns: 30
budget_usd: 1.50
---

# 课件生成 Agent 评测计划

## 评测背景
...

## 评测策略
### 格式约束检查
...
### 常识约束检查
...
### 质量评估
...

## 阈值
| 指标 | 阈值 |
|------|------|
| DR | 0.95 |
| CPR | 0.90 |
| Reward | 0.70 |
```

**EvalPlan 数据模型**（`agent_eval/agent/plan_parser.py`）：

```python
@dataclass
class EvalPlan:
    plan_id: str
    version: str
    target: str
    max_turns: int
    budget_usd: float
    body: str

    def to_system_prompt(self) -> str:
        """将评测计划转换为 Agent system prompt 片段。"""
```

**与其他配置的关系**：
- **替代 pipeline.yaml**：eval_plan.md 中的"评测策略"描述了阶段和评估器组合
- **互补 rule_set.yaml**：eval_plan.md 描述策略层面，rule_set.yaml 定义数据层面
- **互补 prompt 模板**：eval_plan.md 引用评估器名称，Prompt 模板在 `assets/prompts/` 独立管理

### 4.10 Agent 模式配置

```python
class AgentConfig(BaseModel):

```

**CLI 集成**：

```bash
agent-eval agent --plan assets/plans/courseware-eval-v2.md \
  --max-turns 50 --budget-usd 2.00
```

**优先级规则**：CLI 参数 > eval_plan.md frontmatter > 配置文件默认值。
