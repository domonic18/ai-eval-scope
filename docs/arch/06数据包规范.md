# 数据包规范

> 本文档定义执行包与评估结果包的目录结构与字段规范，属于 [01 整体架构设计](./01整体架构设计.md) §二中"基础设施层"的详细展开。

---

## 一、Workspace 总体结构

```
workspace/                              # .gitignore 排除，不入版本控制
├── runs/
│   ├── {run_id}/                       # 如 20260608_143000
│   │   ├── run_manifest.json           # 运行元信息
│   │   ├── packages/                   # 执行包
│   │   │   └── {task_id}/
│   │   ├── results/                    # 评估结果
│   │   │   └── {task_id}/
│   │   └── reports/                    # 聚合报告
│   └── ...
├── index/                              # Web Portal 索引
│   ├── projects.json                   # 项目注册表
│   └── runs_index.json                 # 运行索引（扁平化摘要）
└── cache/                              # 跨运行共享缓存
    └── llm_responses.db
```

## 二、执行包（ExecutionPackage）

每个任务执行后生成一个执行包，存储在 `workspace/runs/{run_id}/packages/{task_id}/`。

### 2.1 目录结构

```
packages/{task_id}/
├── manifest.json          # 数据包元信息
├── task.json              # 原始任务定义
├── output/                # SUT 输出物（Markdown/HTML 文档集）
│   ├── _manifest.json     # 目录清单（仅目录模式时存在）→ 详见 2.2a
│   ├── index.md           # 主文档（或 index.html）
│   ├── chapter_01.md
│   ├── chapter_02.html
│   ├── assets/
│   │   └── image_01.png
│   └── {module_dir}/      # 目录模式：嵌套的模块目录
│       ├── {sub_dir}/
│       │   └── {name}.html
│       └── ...
├── trace.json             # 调用轨迹
├── metrics.json           # 过程指标
└── metadata.json          # SUT 与环境元信息
```

### 2.2 manifest.json

```json
{
  "package_id": "pkg_20260608_143000_math_g7_001",
  "created_at": "2026-06-08T14:30:00Z",
  "task_id": "math_grade7_001",
  "sut_config_id": "agent_a_v1",
  "status": "success"
}
```


**字段说明**：

| 字段 | 说明 |
|------|------|
| `mode` | 固定为 `"directory"`，标识此为目录模式 |
| `root_dir` | 输出文件根目录（相对路径） |
| `total_files` | 文件总数 |
| `file_types` | 按扩展名统计文件数量 |
| `hierarchy_depth` | 最大目录深度 |
| `modules` | 顶层模块列表，每个模块含 name、path、file_count、children |
| `modules[].children` | 模块下所有文件，含相对路径和深度 |

## 三、评估结果包（EvaluationResult）

每个任务评估后生成一个结果包，存储在 `workspace/runs/{run_id}/results/{task_id}/`。

### 3.1 目录结构

```
results/{task_id}/
├── manifest.json          # 评估元信息
├── rule_results.json      # 每条规则的判定结果（含 LLM 溯源字段）
├── scores.json            # 聚合得分
├── evidence/              # 证据目录
│   ├── judge_soft.teaching_logic_20260608_143000.json  # LLM Judge 溯源记录
│   ├── judge_vision.quality_20260608_143005.json       # 多模态评估溯源记录
│   └── ...
├── report.md              # 人类可读报告
└── report.json            # Agent 消费报告
```

### 3.2 manifest.json

```json
{
  "result_id": "eval_20260608_143000_math_g7_001",
  "package_id": "pkg_20260608_143000_math_g7_001",
  "rule_set_version": "courseware_v1.2",
  "evaluated_at": "2026-06-08T14:31:00Z"
}
```

## 五、Workspace 生命周期

### 5.1 创建

每次 `agent-eval` 运行自动在 `workspace/runs/` 下创建以时间戳命名的运行目录。

### 5.2 使用

- **run-only 模式**：仅写入 `packages/`（当前阶段主要由手动打包或简单脚本填充）
- **eval-only 模式**：读取 `packages/`，写入 `results/` + `reports/`（**当前主要使用模式**）
- **pipeline 模式**：完整写入 `packages/` → `results/` → `reports/`（后续执行引擎就绪后启用）

### 5.3 复用

历史运行包可用于：
- 重新评估（更换规则集）
- 人工复评
- 基线对比

### 5.4 清理

Workspace 不入版本控制，可安全删除历史运行。建议保留近期运行用于对比。

### 5.5 Web Portal 索引

为支持 Web Portal 的项目看板和趋势图功能，系统维护一份运行索引：

```
workspace/index/
├── projects.json           # 项目注册表
└── runs_index.json         # 扁平化的运行摘要索引
```

**projects.json 示例**：

```json
[
  {
    "id": "courseware-platform",
    "name": "课件平台 Agent",
    "description": "课件生成 Agent 的评估项目",
    "default_rule_set": "courseware_v1",
    "default_task_set": "courseware_subject_coverage",
    "created_at": "2026-06-08T10:00:00Z",
    "latest_run_id": "20260608_143000",
    "run_count": 5
  }
]
```

**runs_index.json 示例**：

```json
[
  {
    "run_id": "20260608_143000",
    "project_id": "courseware-platform",
    "created_at": "2026-06-08T14:30:00Z",
    "total_samples": 50,
    "dr": 0.94,
    "cpr": 0.88,
    "avg_reward": 1.72,
    "cond_r": 2.05,
    "rule_set_version": "courseware_v1.2",
    "sut_version": "v1.2.0",
    "input_mode": "directory"
  },
  {
    "run_id": "20260607_100000",
    "project_id": "courseware-platform",
    "created_at": "2026-06-07T10:00:00Z",
    "total_samples": 50,
    "dr": 0.92,
    "cpr": 0.85,
    "avg_reward": 1.58,

  }

```

**索引维护策略**：

| 时机 | 操作 |
|------|------|
| 每次 `agent-eval eval` 运行完成 | 自动追加到 `runs_index.json`，更新 `projects.json` 的 `latest_run_id` |
| `agent-eval index` 命令 | 全量扫描 workspace/runs/，重建完整索引 |
| 索引不存在时 | Web Portal 启动时自动触发全量重建 |
