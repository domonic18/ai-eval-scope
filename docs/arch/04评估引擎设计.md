# 评估引擎设计

> 本文档详细阐述评估引擎的设计，包括评估器注册机制、评估器分类体系、评分聚合、指标计算以及评估侧的全部数据模型。属于 [01 整体架构设计](./01整体架构设计.md) §二中"评估引擎"的详细展开。管线编排参见 [02 编排调度层设计](./02编排调度层设计.md)，LLM Judge 实现参见 [05 LLM 模块设计](./05LLM模块设计.md)。

---

## 一、职责边界

评估引擎负责**按照规则对执行结果进行评估与打分**：

| 组件 | 职责 |
|------|------|
| **EvaluatorRegistry** | 评估器注册与工厂创建 |
| **BaseEvaluator 及其子类** | 各类约束检查的具体实现（分类见 §五，清单见 [07](./07评估指标与方法.md)） |
| **ScoreAggregator** | 将约束结果聚合为 Reward Score |
| **MetricsCalculator** | 计算批量指标（DR / CPR / Reward / CondR / Time） |

## 二、评估侧数据模型

### 2.1 枚举定义

```python
class EvalStatus(str, Enum):
    PASS = "pass"
    FAIL = "fail"
    SKIP = "skip"             # 前置阶段失败导致
    ERROR = "error"           # 执行异常

class ConstraintTier(str, Enum):
    HARD_GATE = "hard_gate"       # 硬性门控，失败终止 → S_format = 0（不得分）
    HARD_SCORE = "hard_score"     # 硬性评分，失败归零 → S_common = 0
    SOFT = "soft"                 # 软约束，归一化 [0,1]
    PREFERENCE = "preference"     # 偏好约束，归一化 [0,1]

class EvalMethod(str, Enum):
    RULE = "rule"
    FACT_VERIFY = "fact_verify"
    MATH_VERIFY = "math_verify"
    LLM_JUDGE = "llm_judge"
    LLM_CONSISTENCY = "llm_consistency"
    VISION = "vision"
```

### 2.2 评估结果模型

```python
@dataclass
class ConstraintResult:
    """单项约束检查结果"""
    constraint_id: str                   # 如 "format.response_format"
    name: str
    tier: ConstraintTier
    status: EvalStatus
    score: float = 0.0                   # 归一化后
    raw_score: float | None = None       # 原始得分
    reason: str = ""
    details: dict = field(default_factory=dict)
    duration_ms: float = 0.0
    # LLM Judge 溯源字段（仅 LLM 类评估器填充，详见 05LLM模块设计）
    judge_provider: str | None = None    # 使用的 Provider 名称，如 "deepseek_judge"
    judge_model: str | None = None       # 实际模型 ID，如 "deepseek-chat"
    judge_record_path: str | None = None # 溯源记录文件路径

@dataclass
class StageResult:
    """一个阶段（Stage）的评估结果"""
    stage_id: str                        # "format" | "commonsense" | "quality"
    status: EvalStatus
    constraint_results: list[ConstraintResult] = field(default_factory=list)
    duration_ms: float = 0.0
    gate_passed: bool = True
    category_score: float = 0.0
```

**门控行为说明**：

| 约束层级 | 失败行为 |
|----------|----------|
| `HARD_GATE` | 阶段内立即终止（`fail_fast`），后续阶段全部 `SKIP` |
| `HARD_SCORE` | 阶段内**不中断**其他评估器继续执行，仅标记 `gate_passed=False`；不阻塞后续阶段 |
| `SOFT` / `PREFERENCE` | 全部执行（`continue_all`），不影响阶段通过状态 |

```python
@dataclass
class SampleResult:
    """单个样本的完整评估结果"""
    sample_id: str
    status: EvalStatus
    stage_results: dict[str, StageResult] = field(default_factory=dict)
    s_format: float = 0.0
    s_common: float = 0.0
    s_soft: float = 0.0
    s_pref: float = 0.0
    reward: float = 0.0
    total_duration_ms: float = 0.0
    llm_calls: int = 0
    token_usage: int = 0
    content_hash: str | None = None      # 样本内容指纹（版本标记，用于走势溯源）
```

## 三、EvaluatorRegistry — 评估器注册中心

借鉴 OpenCompass 的装饰器注册模式：

```python
class EvaluatorRegistry:
    def __init__(self):
        self._registry: dict[str, type[BaseEvaluator]] = {}

    def register(self, evaluator_id: str):
        """装饰器：注册评估器类"""
        def decorator(cls):
            self._registry[evaluator_id] = cls
            return cls
        return decorator

    def create(self, evaluator_id: str, params: dict = None) -> BaseEvaluator:
        """工厂方法：根据 ID 创建评估器实例"""
        if evaluator_id not in self._registry:
            raise ValueError(f"未注册的评估器: {evaluator_id}")
        evaluator = self._registry[evaluator_id]()
        if params:
            evaluator.setup(params)
        return evaluator

registry = EvaluatorRegistry()
```

## 五、评估器分类体系

评估器沿**约束层级（ConstraintTier）**与**评估方法（EvalMethod）**两个维度分类，按三阶段级联执行。各评估器的指标定义、检查算法、配置参数、Prompt 模板与知识库依赖统一在 [07 评估指标与方法](./07评估指标与方法.md) 维护；本节仅给出分类维度概览与目录模式的引擎处理策略。

### 5.1 约束层级（决定门控与评分）

| 层级 | 失败行为 | 评分影响 |
|------|----------|----------|
| `HARD_GATE` | 阶段内立即终止（fail_fast），后续阶段全部 SKIP | S_format = 0（不得分） |
| `HARD_SCORE` | 阶段内不中断其他评估器，不阻塞后续阶段 | 对应维度归零（如 S_common = 0） |
| `SOFT` | 全部执行，不影响门控 | 归一化 [0,1] 加权 |
| `PREFERENCE` | 全部执行，不影响门控 | 归一化 [0,1] 加权 |

### 5.2 评估方法（决定实现与降级）

| 方法 | 说明 | 是否依赖 LLM |
|------|------|-------------|
| `RULE` | 纯规则匹配（正则 / 文件属性） | 否 |
| `FACT_VERIFY` | 事实性验证（规则检查 + 误报经 LLM 二次确认 + LLM 语义验证） | 可选 |
| `MATH_VERIFY` | 数学公式验证（括号匹配 / 算术等式 / 符号公式） | 否（枚举保留，当前无评估器使用） |
| `LLM_CONSISTENCY` | 逻辑一致性（LLM 为主；未配置 LLM 时 SKIP 不计分，调用异常降级 Rule-based） | 是（可降级） |
| `LLM_JUDGE` | 纯 LLM 评估（多次采样取中位数；不可用时 SKIP score=0.0 不计分） | 是 |
| `VISION` | 多模态视觉评估（HTML 截图渲染 + 视觉模型判分） | 是（视觉模型） |

### 5.3 阶段 × 方法概览

| 阶段 | 约束层级 | 评估方法 | 覆盖范围 |
|------|----------|----------|----------|
| `format`（格式门控） | HARD_GATE | RULE | 文件格式、HTML 有效性 |
| `commonsense`（常识门控） | HARD_SCORE | FACT_VERIFY / LLM_JUDGE / LLM_CONSISTENCY | 知识准确性、时序正确性、逻辑一致性 |
| `quality`（质量） | SOFT + PREFERENCE | LLM_JUDGE | 教学逻辑、内容多样性、风格、深度、需求满足度 |

> 此外提供 `vision.quality`（VISION 方法）评估器，对渲染截图做视觉质量评估，可按需在规则集中启用。**完整评估器清单及各评估器的指标定义、检查算法见 [07 评估指标与方法](./07评估指标与方法.md)。**

### 5.4 自定义评估器插件

在 `agent_eval/evaluation/evaluators/plugins/` 下创建新 Evaluator，使用 `@registry.register("custom.xxx")` 注册，在 YAML 中引用即可，无需改核心代码。

> **关于 PPTX 评估**：当前评估目标为 Markdown/HTML 文档集。如需支持 PPTX，可通过插件机制扩展 `format.pptx_format` 等评估器，无需修改核心代码。

### 5.5 目录模式评估

当评估目标为包含目录结构的 HTML 文件集时（如课件生成的"大单元学习总导"目录），评估引擎按以下方式处理：

#### 5.5.1 整体流程

1. **目录清单读取**：从 `output/_manifest.json` 读取目录结构信息（由 DirectoryCollector 生成）
2. **逐文件评估**：遍历 manifest 中的所有文件，每个 HTML 文件进入评估管线
3. **按模块聚合**：根据 manifest 中的 modules 信息，支持按模块维度聚合评估结果
4. **全量汇总**：最终将所有文件的评估结果汇总为整体指标

```
output/_manifest.json
       ↓
  ┌────────────────────────────────┐
  │ 24 个 HTML 文件 → 逐文件评估   │
  │ M1 (6文件) → SampleResult × 6  │
  │ M2 (18文件) → SampleResult × 18│
  └──────────────┬─────────────────┘
                 ↓
  ┌────────────────────────────────┐
  │ 按模块聚合                      │
  │ module_results:                │
  │   M1: {reward: 2.1, dr: 1.0}  │
  │   M2: {reward: 1.8, dr: 0.94} │
  └──────────────┬─────────────────┘
                 ↓
  ┌────────────────────────────────┐
  │ 全量汇总 → summary.json        │
  │ DR / CPR / Reward (24样本)     │
  └────────────────────────────────┘
```

#### 5.5.2 评估策略

不同类型评估器在目录模式下采用不同策略：

| 评估器类型 | 评估策略 | 说明 |
|-----------|----------|------|
| **格式评估器（RULE）** | 逐文件 | 每个 HTML 文件独立检查，全部通过才算通过 |
| **常识评估器（RULE/FACT）** | 可配置 | 默认逐文件；部分检查（如时序正确性）可合并同一模块内文件后评估 |
| **LLM Judge** | 可配置粒度 | 支持 `package`（整体）/ `module`（按模块）/ `file`（逐文件）三种粒度，默认 `package` 级 |

粒度通过评估器参数 `directory_granularity` 控制：

```yaml
# pipeline.yaml 中评估器粒度配置
- name: soft.teaching_logic
  params:
    template_id: pedagogical_logic
    directory_granularity: module    # 按模块评估（每个模块调用一次 LLM）
```

#### 5.5.3 模块级结果

对于目录模式任务，`rule_results.json` 中的 ConstraintResult 增加可选的 `module_results` 字段，记录各模块的评估子结果。详见 [06 数据管理与配置规范](./06数据管理与配置规范.md) §三.4。

> 对于目录模式任务，`SampleResult.sample_id` 格式为 `{task_id}/{relative_path}`，确保每个文件在报告中可追溯。

### 5.6 规则启用控制（`enabled`）

评估器的实际执行集合由 RuleSet 中 `Rule.enabled` 决定，引擎在构建评估管线时过滤：

- `enabled: false` 的规则（评估器）从管线移除，不参与评估与计分；
- 未在 RuleSet 中显式声明的评估器默认启用（registry 全集生效）。

实现在 `evaluation/engine.py` 的 `apply_rule_set_params()`：遍历 `rule_set.rules`，跳过 `enabled=false` 项。可在 `assets/rules/default_rule_set.yaml` 中按需置 `enabled: false` 关闭某评估器（如临时关闭某 commonsense 检查），无需改代码。

## 六、评分聚合（ScoreAggregator）

### 6.1 Reward 公式

```
Reward = (format_pass × S_format + commonsense_pass × S_common
          + w3 × S_soft + w4 × S_pref)
         / (format_pass + commonsense_pass + w3 + w4)
```

> 各维度 ∈ [0,1]，Reward 归一化到 **[0,1]、无负值**；format 失败的严重性由 **DR（格式通过率）+ fail-fast 短路** 体现（详见 [07 §5.2](./07评估指标与方法.md)）。权重默认值见 `ScoreAggregationWeights`（`format_pass=1` / `format_fail=0` / `commonsense_pass=1` / `w3=w4=1`）。

| 符号 | 含义 | 取值 |
|------|------|------|
| S_format | 格式约束得分 | +1 / 0（`format_pass=1` / `format_fail=0`） |
| S_common | 常识约束得分 | +1 / 0（`commonsense_pass=1`） |
| S_soft | 软约束得分 | [0, 1] 加权平均 |
| S_pref | 偏好约束得分 | [0, 1] 加权平均 |
| w3, w4 | 乘数 | 默认 1.0，可配置 |

### 6.2 SampleScore

```python
@dataclass
class SampleScore:
    sample_id: str
    s_format: float
    s_common: float
    s_soft: float
    s_pref: float
    reward: float = 0.0    # 由 ScoreAggregator 按上式归一化计算后写入（非 property）
```

### 6.3 ScoreAggregator 实现

```python
class ScoreAggregator:
    def __init__(self, w3: float = 1.0, w4: float = 1.0,
                 soft_weights: dict | None = None, pref_weights: dict | None = None):
        self.w3 = w3
        self.w4 = w4
        # 默认权重取自 SCORE_AGGREGATION_WEIGHTS（config/evaluation.py）
        self.soft_weights = soft_weights or {
            "soft.teaching_logic": 0.5, "soft.content_diversity": 0.5,
        }
        self.pref_weights = pref_weights or {
            "pref.style_preference": 0.33, "pref.depth_preference": 0.33,
            "pref.request_fulfillment": 0.34,
        }

    def aggregate(self, result: SampleResult) -> SampleScore:
        s_format = self._format(result)
        s_common = self._commonsense(result)
        s_soft = self._weighted(result, "quality", self.soft_weights)
        s_pref = self._weighted(result, "quality", self.pref_weights)
        # 归一化到 [0,1]：除以满分基数（format_pass + commonsense_pass + w3 + w4）
        denom = 1 + 1 + self.w3 + self.w4
        reward = (s_format + s_common + self.w3 * s_soft + self.w4 * s_pref) / denom
        return SampleScore(
            sample_id=result.sample_id, s_format=s_format, s_common=s_common,
            s_soft=s_soft, s_pref=s_pref, reward=reward,
        )

    def _format(self, r: SampleResult) -> float:
        s = r.stage_results.get("format")
        if not s or s.status == EvalStatus.SKIP: return 0.0
        return 1.0 if s.status == EvalStatus.PASS else 0.0   # format_fail（默认 0，不再用 -3）

    def _commonsense(self, r: SampleResult) -> float:
        s = r.stage_results.get("commonsense")
        if not s or s.status in (EvalStatus.SKIP, EvalStatus.FAIL): return 0.0
        return 1.0

    def _weighted(self, r: SampleResult, stage_id: str, weights: dict) -> float:
        s = r.stage_results.get(stage_id)
        if not s or s.status == EvalStatus.SKIP: return 0.0
        # SKIP 约束（LLM 不可用等）不计入得分，避免 Reward 虚高
        wsum = sum(weights[c.constraint_id] * c.score
                   for c in s.constraint_results
                   if c.status != EvalStatus.SKIP and c.constraint_id in weights)
        wtotal = sum(weights.values())
        return wsum / wtotal if wtotal > 0 else 0.0
```

## 七、指标计算（MetricsCalculator）

### 7.1 核心指标

| 指标 | 含义 | 计算 | 默认阈值 |
|------|------|------|----------|
| **DR** | 交付率 | 格式通过数 / 总数 | ≥ 0.95 |
| **CPR** | 常识通过率 | 常识通过数 / 总数 | ≥ 0.90 |
| **Reward** | 全样本平均 Reward | Σ reward / n | ≥ 0.70 |
| **avg_soft** | 平均内容质量分 | Σ s_soft / n（独立指标，不混入 Reward） | — |
| **avg_pref** | 平均用户偏好分 | Σ s_pref / n（独立指标，不混入 Reward） | — |
| **CondR** | 条件奖励 | 通过门控样本的 Reward 均值 | — |
| **Time** | 平均完成时间 | Σ duration / n | — |

### 7.2 MetricsReport

```python
@dataclass
class MetricsReport:
    total_samples: int = 0
    dr: float = 0.0
    cpr: float = 0.0
    avg_reward: float = 0.0
    cond_r: float = 0.0
    avg_time_ms: float = 0.0
    avg_soft: float = 0.0                                 # 平均内容质量分（独立指标）
    avg_pref: float = 0.0                                 # 平均用户偏好分（独立指标）
    llm_skipped: int = 0                                  # LLM 不可用而 SKIP 的样本数
    sample_scores: list[SampleScore] = field(default_factory=list)  # 逐样本得分（含 reward）
    thresholds: dict = field(default_factory=dict)        # 使用的阈值快照
    failure_breakdown: dict[str, int] = field(default_factory=dict)
```

### 7.3 MetricsCalculator 实现

```python
class MetricsCalculator:
    def compute(self, results: list[SampleResult]) -> MetricsReport:
        total = len(results)
        if total == 0: return MetricsReport()

        fmt_pass = sum(1 for r in results
                       if r.stage_results.get("format", StageResult(status=EvalStatus.FAIL)).gate_passed)
        com_pass = sum(1 for r in results
                       if r.stage_results.get("format").gate_passed
                       and r.stage_results.get("commonsense", StageResult(status=EvalStatus.FAIL)).gate_passed)
        gated = [r for r in results if self._passed_gates(r)]

        return MetricsReport(
            total_samples=total,
            dr=fmt_pass / total,
            cpr=com_pass / total,
            avg_reward=sum(r.reward for r in results) / total,
            avg_soft=sum(r.s_soft for r in results) / total,
            avg_pref=sum(r.s_pref for r in results) / total,
            llm_skipped=self._llm_skipped(results),
            cond_r=(sum(r.reward for r in gated) / len(gated)) if gated else 0.0,
            avg_time_ms=sum(r.total_duration_ms for r in results) / total,
            sample_scores=[self._aggregator.aggregate(r) for r in results],
            failure_breakdown=self._breakdown(results),
        )

    def _passed_gates(self, r: SampleResult) -> bool:
        f = r.stage_results.get("format")
        c = r.stage_results.get("commonsense")
        return f and f.gate_passed and c and c.gate_passed

    def _breakdown(self, results: list[SampleResult]) -> dict[str, int]:
        b = {}
        for r in results:
            for sr in r.stage_results.values():
                for cr in sr.constraint_results:
                    if cr.status == EvalStatus.FAIL:
                        b[cr.constraint_id] = b.get(cr.constraint_id, 0) + 1
        return b
```
