# 数据集下载设计

> 本文档定义「评测数据集下载」功能的设计方案：通过 CLI 从 HuggingFace / ModelScope 下载公开评测数据集到本地 workspace，供后续评估流水线消费。属于 [06 数据管理与配置规范](./06数据管理与配置规范.md) 中「运行数据」维度的扩展——为 workspace 新增 `datasets/` 子目录约定与数据获取入口。
>
> 设计参考了 OpenCompass 的多源下载实现（`docs/reference/03调研报告_opencompass.md`），但在触发方式上做了关键差异：OpenCompass 是**惰性触发**（评测运行时按需下载，无独立下载命令），本系统采用**显式 CLI 命令**（`agent-eval dataset download`），更契合「评测数据与评测执行解耦」的 workspace 模型。

---

## 一、总览

### 1.1 需求背景

- 评测系统需要消费外部公开评测数据集（如 ceval、gsm8k、mmlu、mbpp 等）作为黄金样本或被测输入。
- 国内网络环境下 HuggingFace 直连不稳定，需支持 ModelScope 源与 HF 镜像（`hf-mirror.com`）。
- 数据集体积大、变动少，应与代码仓库解耦，落在 `workspace/`（gitignored）下，而非随包发布的 `assets/`。

### 1.2 设计目标

| 目标 | 说明 |
|------|------|
| **双源支持** | 同时支持 HuggingFace（`huggingface_hub`）与 ModelScope（`modelscope`）两个下载源，可按命令行参数或环境变量切换 |
| **可配置路径** | 支持命令行 `--output` 指定下载目录；默认落盘到 `workspace/datasets/{name}/`，复用现有 workspace 定位机制（`WORKSPACE_DIR`） |
| **可追溯** | 每次下载落盘时写入 `_dataset_manifest.json`，记录来源、版本、下载时间，便于复现与审计 |
| **依赖可选** | `huggingface_hub` / `modelscope` 作为可选依赖（`datasets` extra），核心包不强制安装；缺失时给出友好的安装指引 |
| **风格一致** | 新增的枚举、异常、模块结构、CLI 命令组织、日志、测试，严格遵循现有代码约定 |

### 1.3 与 OpenCompass 的关键差异

| 维度 | OpenCompass | 本系统 |
|------|-------------|--------|
| 触发方式 | 惰性触发（`load()` 内按 `DATASET_SOURCE` 分派） | 显式 CLI 命令（`agent-eval dataset download`） |
| 默认目录 | `~/.cache/opencompass/`（`COMPASS_DATA_CACHE`） | `workspace/datasets/`（`WORKSPACE_DIR`），与运行产物同根 |
| 数据集标识 | 仅记录三元 id（`ms_id`/`hf_id`/`local`），不存储到固定目录 | id + 落盘目录 + manifest，下载结果可直接被后续模块引用 |
| 版本支持 | 不支持 `revision` | 支持 `--revision`（HF commit / MS 版本） |
| 源切换 | 全局环境变量 `DATASET_SOURCE` | **命令行参数优先**（`--source`），环境变量 `AGENT_EVAL_DATASET_SOURCE` 作为兜底默认 |

> 设计取舍说明：OpenCompass 的惰性触发与其「configs/*.py 驱动一次跑」的工作流绑定；本系统的 workspace 模型强调「先准备数据 → 再打包/评估」，下载应是独立的、可重跑的、可审计的步骤，故采用显式命令。

### 2.2 使用示例

```bash

agent-eval dataset download ceval --source modelscope
```

## 四、详细设计

### 4.1 数据集注册表（`datasets/registry.py`）— 数据驱动

参考 OpenCompass 的两份数据：`DATASETS_MAPPING`（hf_id/ms_id 地址，`datasets_info.py`）与 `dataset-index.yml`（name/category/paper 元数据）。本系统**将两者合并为一份随包 yaml 索引**，registry 首次访问时加载并经 JSON Schema 校验——**维护数据集地址只需编辑 yaml，无需改代码**。

**索引文件**：`agent_eval/assets/datasets/dataset_index.yaml`（初版精选 13 个常用集，移植自 OpenCompass 两份源文件交叉匹配而来），结构如下：

```yaml
# agent_eval/assets/datasets/dataset_index.yaml
version: "1.0"
description: "评测数据集索引"
datasets:
  - id: ceval                      # 唯一标识（CLI 传入名 / 目录名 / 查表键）
    name: C-Eval                   # 展示名（来自 dataset-index.yml）
    category: Examination          # 能力类别（来自 dataset-index.yml）
    paper: https://arxiv.org/...   # 论文链接（来自 dataset-index.yml）
    description: 中文基础模型评测（52 学科）
    default_source: modelscope     # auto / huggingface / modelscope
    sources:
      hf_id: opencompass/ceval-exam   # 来自 DATASETS_MAPPING
      ms_id: opencompass/ceval-exam   # 来自 DATASETS_MAPPING
  - id: gsm8k
    ...
```

**加载与 API**（`agent_eval/datasets/registry.py` 节选）：

```python
@dataclass(frozen=True)
class DatasetEntry:
    id: str                        # 唯一标识，如 "ceval"
    name: str                      # 展示名，如 "C-Eval"
    hf_id: str | None = None
    ms_id: str | None = None
    description: str = ""
    default_source: str = "auto"
    category: str = ""             # 来自 dataset-index.yml
    paper: str = ""                # 来自 dataset-index.yml

    def get_id(self, source: str) -> str | None:
        if source == "huggingface":
            return self.hf_id
        if source == "modelscope":
            return self.ms_id
        return None


@lru_cache(maxsize=1)
def _load_index() -> dict[str, DatasetEntry]:
    """经 ConfigLoader + dataset_index_schema.json 校验后构建 id → entry 映射。"""
    schema = get_schema_path("dataset_index_schema.json")
    data = ConfigLoader.load_and_validate(_INDEX_PATH, schema)
    ...


def lookup(dataset_id: str) -> DatasetEntry | None:
    """按 id 查注册表，未命中返回 None。"""

def list_datasets() -> dict[str, DatasetEntry]:
    """返回全部注册数据集（供 `agent-eval dataset list` 渲染表格）。"""
```

> **设计要点**：
> - **数据驱动**：增删/更新数据集 = 编辑 yaml；`_load_index` 用 `lru_cache` 首次加载并缓存，`dataset_index_schema.json` 校验格式（id 命名、source 枚举、必填字段）。
> - **`id` vs `name`**：`id` 是 CLI 传入的标识（目录名/查表键），`name` 是展示名。未注册的数据集可直接传完整 repo id，registry 视其为「id = name = repo id」的临时 entry。
> - **双源地址合并**：yaml 的 `sources.hf_id`/`ms_id` 留空（null）表示该源暂不可用，可后续补充——这正是「维护获取地址」的入口。
> - OpenCompass 的 `DATASETS_URL`（自建 OSS 下载表）本系统不引入；如未来需离线分发，可在 yaml 增补 `oss` 子字段并复刻其 `download_and_extract_archive`（`fileio.py:509`）。

### 4.2 统一下载器抽象（`datasets/downloader.py`）

```python
# agent_eval/datasets/downloader.py
"""数据集下载器 — HuggingFace / ModelScope 双源实现。

惰性导入第三方库，缺失时抛 DatasetError 并给出安装指引
（参照 agent_eval/observability/render.py 的 Playwright 懒加载范式）。
"""
from __future__ import annotations

from abc import ABC, abstractmethod
from pathlib import Path

from agent_eval.core.exceptions import DatasetDownloadError
from agent_eval.core.types import DatasetSource


class DatasetDownloader(ABC):
    """下载器抽象基类。"""

    @abstractmethod
    def download(
        self,
        repo_id: str,
        target: Path,
        revision: str | None = None,
        token: str | None = None,
    ) -> Path:
        """下载数据集到 target 目录，返回目标路径。"""
        ...


class HuggingFaceDownloader(DatasetDownloader):
    """HuggingFace 下载器，基于 huggingface_hub.snapshot_download。

    用 local_dir 落盘到指定目录（而非默认 ~/.cache/huggingface），
    镜像通过 HF_ENDPOINT 环境变量透传（不在代码硬编码，对齐 OpenCompass 做法）。
    """

    def download(self, repo_id, target, revision=None, token=None):
        try:
            from huggingface_hub import snapshot_download
        except ImportError as e:  # pragma: no cover
            raise DatasetDownloadError(
                "huggingface_hub 未安装。请执行: uv sync --extra datasets"
            ) from e

        target.mkdir(parents=True, exist_ok=True)
        snapshot_download(
            repo_id=repo_id,
            repo_type="dataset",
            local_dir=str(target),
            revision=revision,          # None → 最新
            token=token,                # None → 读 HF_TOKEN
        )
        return target


class ModelScopeDownloader(DatasetDownloader):
    """ModelScope 下载器。

    整仓下载原始文件用 dataset_snapshot_download；
    仅需 Dataset 对象可改用 MsDataset.load（缓存自管，落盘格式不同）。
    """

    def download(self, repo_id, target, revision=None, token=None):
        try:
            from modelscope.hub.snapshot_download import dataset_snapshot_download
        except ImportError as e:  # pragma: no cover
            raise DatasetDownloadError(
                "modelscope 未安装。请执行: uv sync --extra datasets"
            ) from e

        target.mkdir(parents=True, exist_ok=True)
        dataset_snapshot_download(
            dataset_name=repo_id,
            cache_dir=str(target.parent),
            revision=revision,
        )
        return target


def get_downloader(source: DatasetSource | str) -> DatasetDownloader:
    """工厂：按 source 返回下载器实例。"""
    source = DatasetSource(source) if not isinstance(source, DatasetSource) else source
    if source is DatasetSource.HUGGINGFACE:
        return HuggingFaceDownloader()
    if source is DatasetSource.MODELSCOPE:
        return ModelScopeDownloader()
    raise DatasetDownloadError(f"不支持的下载源: {source}")
```

### 4.3 编排器（`datasets/manager.py`）

```python
# agent_eval/datasets/manager.py
"""DatasetManager — 下载编排：解析 → 选源 → 下载 → 落盘 → 写 manifest。"""
from __future__ import annotations

import json
import os
from datetime import UTC, datetime
from pathlib import Path

from agent_eval.config.paths import paths
from agent_eval.core.exceptions import DatasetError, DatasetNotFoundError
from agent_eval.core.logging import get_logger
from agent_eval.core.types import DatasetSource
from agent_eval.datasets.downloader import get_downloader
from agent_eval.datasets.registry import DatasetEntry, lookup

logger = get_logger(__name__)


class DatasetManager:
    """数据集下载编排器。"""

    def download(
        self,
        name: str,
        source: str | None = None,
        output: Path | None = None,
        revision: str | None = None,
        token: str | None = None,
        force: bool = False,
    ) -> Path:
        """下载单个数据集，返回落盘目录。"""
        # 1. 解析 entry（注册表命中 / 视为完整 repo id）
        entry = lookup(name)
        if entry is None:
            entry = DatasetEntry(name=name, hf_id=name, ms_id=name,
                                 description="(用户指定 repo id)")

        # 2. 解析 source：参数 > 环境变量 > entry.default_source
        resolved_source = self._resolve_source(source, entry)

        # 3. 取对应源 id
        repo_id = entry.get_id(resolved_source)
        if not repo_id:
            raise DatasetNotFoundError(
                f"数据集 '{name}' 在 {resolved_source} 源上未提供 id"
            )

        # 4. 解析目标目录
        target = self._resolve_target(name, output)
        if target.exists() and any(target.iterdir()) and not force:
            logger.info("dataset.exists", name=name, target=str(target))
            return target
        if force and target.exists():
            import shutil; shutil.rmtree(target)

        # 5. 下载
        logger.info("dataset.download.start", name=name, source=resolved_source, repo_id=repo_id)
        downloader = get_downloader(resolved_source)
        downloader.download(repo_id, target, revision=revision, token=token)

        # 6. 写 manifest
        self._write_manifest(target, entry, resolved_source, repo_id, revision)
        logger.info("dataset.download.done", name=name, target=str(target))
        return target

    def _resolve_source(self, source: str | None, entry: DatasetEntry) -> str:
        if source:
            return source
        env = os.environ.get("AGENT_EVAL_DATASET_SOURCE")
        if env:
            return env
        return entry.default_source  # 通常 "auto"，由 CLI 兜底转 ms

    def _resolve_target(self, name: str, output: Path | None) -> Path:
        if output:
            return Path(output)
        return paths.default_workspace / "datasets" / name

    def _write_manifest(self, target, entry, source, repo_id, revision):
        manifest = {
            "name": entry.name,
            "source": source,
            "repo_id": repo_id,
            "revision": revision,
            "description": entry.description,
            "downloaded_at": datetime.now(UTC).isoformat(),
        }
        (target / "_dataset_manifest.json").write_text(
            json.dumps(manifest, ensure_ascii=False, indent=2), encoding="utf-8"
        )
```

> **目录命名说明**：`target` 默认用预置 `name`（如 `ceval`）；若用户传完整 repo id（`opencompass/ceval-exam`），目录名含 `/` 不合法，CLI 层应在调用前对 name 做规范（取最后一段或替换 `/` 为 `_`）。

### 6.2 惰性导入策略

- `huggingface_hub` / `modelscope` **不写入** `dependencies`（核心依赖），仅在 `datasets/` 子包内**惰性 import**。
- 失败时抛 `DatasetDownloadError`，消息包含 `uv sync --extra datasets` 指引（参照 `observability/render.py` 的 Playwright 范式）。
- 与 OpenCompass 一致：OpenCompass 的 `modelscope` 同样是**纯惰性导入、不声明依赖**（`ai-opencompass/setup.py` requirements 中无 modelscope），避免重依赖污染。

## 七、镜像与认证（环境变量）

**不在代码中硬编码任何端点或凭据**，全部通过环境变量透传给底层库：

| 环境变量 | 作用 | 透传目标 | 说明 |
|----------|------|----------|------|
| `HF_ENDPOINT` | HF 镜像端点 | `huggingface_hub` | 国内设 `https://hf-mirror.com` |
| `HF_TOKEN` / `HUGGING_FACE_HUB_TOKEN` | HF 认证 | `huggingface_hub` | 私有/受限数据集 |
| `MODELSCOPE_API_TOKEN` | MS 认证 | `modelscope` | 私有数据集 |
| `AGENT_EVAL_DATASET_SOURCE` | 本系统默认源 | `DatasetManager` | `--source` 未指定时兜底 |
| `WORKSPACE_DIR` / `AGENT_EVAL_WORKSPACE` | workspace 根 | `paths.default_workspace` | 已有机制，复用 |

> 镜像方案对齐 OpenCompass FAQ（`ai-opencompass/docs/zh_cn/get_started/faq.md:102-104`）的官方推荐：`HF_ENDPOINT=https://hf-mirror.com ...`。

## 十、待决问题（后续迭代）

1. **数据集格式归一**：不同源/数据集的文件格式各异（jsonl / parquet / 多 subset）。是否需要一个 `DatasetLoader` 把它们统一成 `ExecutionPackage` 可消费的标准结构？初版不引入，仅做「原始下载」。
2. **`list` / `info` 子命令**：列出注册表与本地已下载数据集状态，作为易用性增强。
3. **校验**：是否引入文件校验（MD5/SHA256）保证下载完整性？OpenCompass 的 OSS 模式有 MD5 校验（`fileio.py:420`），HF/MS 模式无。初版依赖底层库自身的完整性保证，后续可补。
4. **国内 OSS 源**：若 HF/MS 均不可用，是否引入自建 OSS 兜底源（复刻 OpenCompass `download_and_extract_archive`）？视实际需求决定。
