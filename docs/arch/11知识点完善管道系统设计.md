# 知识点完善管道系统设计

> 本文档设计 `agent_eval/knowledge_pipeline/` 模块——将现有散乱的 `scripts/` 脚本重构为统一的、可扩展的、CLI 集成的知识点完善管道系统。支撑 [02 知识点数据集完善方案](../plan/02知识点数据集完善方案.md) 的后续工作。

---

## 一、设计背景

### 1.1 现有问题

| 问题 | 具体表现 | 影响 |
|------|----------|------|
| **规范性差** | 脚本在 `scripts/`（不在 `agent_eval` 包内）；无类型 schema；ADAPTERS 手动注册 | 无法被包内模块复用 |
| **抽象不足** | 无 adapter 基类/协议；函数签名不统一（arc `(data_dir, limit)` vs cmmlu `(data_dir, limit, subjects)`）；main 硬编码 `if dataset == "cmmlu"` 分发 | 新数据源需改核心代码 |
| **复用性低** | `parse_dataset.py` 不被包内任何模块 import；`extract_knowledge.py` 与 parse_dataset 紧耦合 | 功能孤立 |
| **无写入 API** | `KnowledgeBaseManager` 只有 `load()`（纯读取），无合并/校验 | 提取产物→knowledge 的合并全靠手工 |
| **元数据未消费** | `dataset_index.yaml` 的 `knowledge_mapping`（subjects/levels/extractable）无代码读取 | 标注是"死数据" |
| **无 CLI** | 用户需手动 `cd evaluator && uv run python ../scripts/xxx.py` | 易用性差 |

### 1.2 可复用的现有抽象

| 现有模块 | 抽象 | 质量 | 复用方式 |
|----------|------|------|----------|
| `datasets/downloader.py` | `DatasetDownloader` ABC + 工厂 | 高 | Pipeline 下载步骤直接调用 |
| `datasets/manager.py` | `DatasetManager.download()` | 高 | 一行完成下载编排 |
| `datasets/registry.py` | `DatasetEntry.knowledge_mapping` | 中（有数据无消费） | DataSource 消费此元数据 |
| `knowledge/manager.py` | `KnowledgeBaseManager.load()` | 中（只读） | 不改，Merger 独立写入 |
| `llm/factory.py` | `LLMClientFactory.create()` | 高 | LLM 提取器直接用 |
| `rules/manager.py` | `RuleSetManager`（apply/diff/rollback） | 高 | **范式参考**（不导入） |

## 三、核心 ABC 接口

### 3.1 DataSource（统一数据源）

```python
class DataSource(ABC):
    """数据源抽象基类。用 kind 区分两类源。"""

    kind: ClassVar[str]  # "questions"（评测题）或 "raw_items"（结构化表）

    @abstractmethod
    def read(self, limit: int | None = None, **kwargs) -> list[Question] | list[dict]:
        """读取数据源。

        Returns:
            kind="questions" 时返回 list[Question]；
            kind="raw_items" 时返回 list[dict]（原始结构化数据）。
        """

    @classmethod
    def from_entry(cls, entry: DatasetEntry) -> DataSource:
        """从 dataset_index.yaml 的 DatasetEntry 构造（消费 knowledge_mapping）。

        默认实现：用 entry.id 查 registry，用 knowledge_mapping.subjects 过滤。
        """
        ...
```

### 3.2 Extractor（LLM 提取，消费 questions）

```python
class Extractor(ABC):
    """LLM 提取器抽象基类。"""

    field: ClassVar[str]  # "misconceptions" / "constants"

    @abstractmethod
    def extract(self, questions: list[Question], **kwargs) -> ExtractedBatch:
        """从评测题列表提取知识点。

        内部调用 LLM（复用 LLMClientFactory + client.chat）。
        """
```

### 3.3 Converter（结构化转换，消费 raw_items）

```python
class Converter(ABC):
    """结构化转换器抽象基类。"""

    field: ClassVar[str]  # "constants" / "domain_facts"

    @abstractmethod
    def convert(self, raw_items: list[dict], **kwargs) -> ExtractedBatch:
        """从结构化数据转换知识点（纯数据转换，无 LLM）。

        如：周期表 JSON → constants（Kelvin→℃，符号→中文映射）。
        """
```

## 四、Pydantic 数据模型

```python
class Choice(BaseModel):
    label: str
    text: str

class Answer(BaseModel):
    label: str
    text: str

class Question(BaseModel):
    """统一题目模型（parse_dataset.py 产出的 JSONL 格式的类型化版本）。"""
    id: str
    question: str
    choices: list[Choice]
    answer: Answer
    source: str
    source_subset: str | None = None

    @property
    def distractors(self) -> list[str]:
        """干扰项文本列表（排除正确答案）。"""
        return [c.text for c in self.choices if c.label != self.answer.label]

class ExtractedItem(BaseModel):
    """单条提取产物（带来源追溯）。"""
    field: str           # constants / misconceptions / domain_facts
    data: dict           # {name, value, ...} 或 {pattern, correct, ...}
    source: str          # 题目 id 或数据源 id（追溯）

class ExtractedBatch(BaseModel):
    """一批提取产物。"""
    field: str
    items: list[ExtractedItem]
    source_dataset: str  # 数据源名（arc / cmmlu / periodic_table）
    extractor: str       # 提取器/转换器类名

class KnowledgePatch(BaseModel):
    """可合并到 knowledge yaml 的补丁。"""
    field: str           # constants / misconceptions / domain_facts
    items: list[dict]    # 直接的 dict（合并到 yaml 对应字段）
    source: str          # 来源标注

    @classmethod
    def from_batch(cls, batch: ExtractedBatch) -> KnowledgePatch:
        """从 ExtractedBatch 转换（提取 data 字段，丢弃元数据）。"""

    @classmethod
    def from_yaml(cls, path: Path) -> KnowledgePatch:
        """从待审核 YAML 文件加载（extract/convert 命令的输出）。"""
```

## 五、注册机制（插件式扩展）

```python
# registry.py
_SOURCES: dict[str, type[DataSource]] = {}
_EXTRACTORS: dict[str, type[Extractor]] = {}
_CONVERTERS: dict[str, type[Converter]] = {}


def register_source(name: str):
    """装饰器：注册数据源。新数据源只需在 sources/ 下新建文件并标注此装饰器。"""
    def decorator(cls):
        _SOURCES[name] = cls
        return cls
    return decorator


def register_extractor(name: str):
    """装饰器：注册 LLM 提取器。"""
    def decorator(cls):
        _EXTRACTORS[name] = cls
        return cls
    return decorator


def register_converter(name: str):
    """装饰器：注册结构化转换器。"""
    def decorator(cls):
        _CONVERTERS[name] = cls
        return cls
    return decorator


def get_source(name: str) -> DataSource:
    """工厂：按名获取数据源实例。"""
    if name not in _SOURCES:
        raise KnowledgePipelineError(
            f"未注册的数据源: {name}，已注册: {list(_SOURCES)}"
        )
    return _SOURCES[name]()


def get_extractor(field: str) -> Extractor:
    """工厂：按 field（misconceptions/constants）获取提取器。"""
    ...


def get_converter(name: str) -> Converter:
    """工厂：按名获取转换器（periodic_table / nist_codata）。"""
    ...


# __init__.py 触发内置注册
def discover_builtin():
    """导入内置模块，触发 @register_* 装饰器注册。"""
    import agent_eval.knowledge_pipeline.sources.eval_sources  # noqa: F401
    import agent_eval.knowledge_pipeline.sources.table_sources  # noqa: F401
    import agent_eval.knowledge_pipeline.extractors.llm_extractor  # noqa: F401
    import agent_eval.knowledge_pipeline.converters.periodic_table  # noqa: F401
```

**扩展新数据源**：只需在 `sources/` 新建文件 + `@register_source("xxx")`，不改核心代码。

## 六、KnowledgeMerger（写入/合并 API）

**新建 Merger**（不扩展 KnowledgeBaseManager，保持后者纯读单一职责）：

```python
class KnowledgeMerger:
    """知识点合并器——将提取/转换产物合并到 knowledge yaml。

    范式参考 rules/manager.py 的 RuleSetManager（apply/归档/写盘）。
    """

    def __init__(self, knowledge_dir: Path | None = None):
        self.knowledge_dir = knowledge_dir or paths.assets_dir / "knowledge"

    def merge(
        self,
        patch: KnowledgePatch,
        subject: str,
        strategy: str = "skip",  # "skip"（重名跳过）/ "replace"（覆盖）
        dry_run: bool = False,   # 只报告不写盘
    ) -> dict:
        """合并 patch 到 <subject>.yaml 的对应字段。

        Returns:
            {added: int, skipped: int, replaced: int, total: int}
        """
        # 1. 读现有 <subject>.yaml
        # 2. 按 field 追加 patch.items
        # 3. 去重（键：constants→name, misconceptions→pattern）
        # 4. dry_run=False 时写回 yaml
        # 5. 返回统计

    def list_field(self, subject: str, field: str) -> list[dict]:
        """列出某学科某字段的现有条目。"""
```

### 去重键

| 字段 | 去重键 | 冲突策略 |
|------|--------|----------|
| constants | `name` | skip / replace |
| misconceptions | `pattern` | skip / replace |
| domain_facts | 按学科子键（如 `common_elements.symbol`） | skip / replace |

## 八、Pipeline 编排

```python
class KnowledgePipeline:
    """端到端管道：download → read → extract/convert → review → merge。"""

    def run(
        self,
        source_name: str,
        field: str,
        subject: str,
        limit: int | None = None,
        apply: bool = False,  # False=只输出待审核 YAML，True=自动合并
    ) -> Path:
        """
        1. get_source(source_name) → source
        2. source.read(limit) → questions 或 raw_items
        3. 按 source.kind 分支：
           - questions → get_extractor(field).extract(questions) → batch
           - raw_items → get_converter(source_name).convert(raw_items) → batch
        4. batch → 写 workspace/knowledge_extract/<subject>_<field>.yaml（待审核）
        5. apply=True 时 → KnowledgeMerger.merge(patch, subject)
        """
```

### 使用示例

```bash
# 周期表 → chemistry constants（116 条）
agent-eval knowledge convert --source periodic_table --field constants

# arc → misconceptions（58 条/20 题）
agent-eval knowledge extract --source arc --field misconceptions --limit 20

# 合并到 knowledge yaml
agent-eval knowledge merge --input workspace/knowledge_extract/chemistry_constants.yaml --subject chemistry

# 端到端（提取+自动合并）
agent-eval knowledge extract --source arc --field misconceptions --limit 20 --apply
```
