# 配置规范

> 本文档定义系统所有配置文件的格式与 Schema，属于 [01 整体架构设计](./01整体架构设计.md) §二中"基础设施层"的详细展开。评估侧数据模型参见 [04 评估引擎设计](./04评估引擎设计.md)，执行侧数据模型参见 [03 执行引擎设计](./03执行引擎设计.md)。

## 一、配置文件总览

| 配置文件 | 存放位置 | 作用 |
|----------|----------|------|
| `pipeline.yaml` | `assets/rules/` | 管线配置：阶段、评估器、权重、阈值（**被 Markdown 评测计划替代**） |
| **`eval_plan.md`** | `assets/plans/` | **Markdown 评测计划**：YAML frontmatter + Markdown 正文，定义评测背景、策略、阈值与预算 |
| `rule_set.yaml` | `assets/rules/{scenario}/` | 规则集：具体规则定义 |
| `task_set.yaml` | `assets/tasks/{scenario}/` | 任务集：输入数据定义 |
| `project.yaml` | `assets/projects/` | 项目配置：评估运行分组 |
| `sut_config.yaml` | CLI 参数或 `assets/` | 被测 Agent 连接配置 |
| `llm_config.yaml` | CLI 参数或 `assets/` | LLM Provider 配置 |
| Prompt 模板 | `assets/prompts/` | LLM Judge 的评估模板 |
| JSON Schema | `assets/schemas/` | 配置文件校验 Schema |

## 二、管线配置（pipeline.yaml）

```yaml
# assets/rules/_default.yaml
pipeline:
  name: courseware-eval-v1
  version: "1.0"
  description: "课件生成Agent标准评估管线"

  # Reward 公式参数 → 详见 06 评分与指标设计
  reward_weights:
    w3: 1.0    # 软约束乘数
    w4: 1.0    # 偏好约束乘数

  # 级联阶段定义 → 详见 03 管线引擎设计
  stages:
    - id: format
      name: "格式约束检查"
      short_circuit_policy: fail_fast
      evaluators:
        - name: format.response_format
          params:
            allowed_formats: ["md", "html"]
        - name: format.document_count
          params:
            min_documents: 1
            max_documents: 20
        - name: format.structure_compliance
          params:
            max_heading_depth: 4          # 标题最大层级
            require_toc: false            # 是否要求目录
        - name: format.html_validity
          params:
            check_html_only: true         # 仅检查 HTML 文件，Markdown 跳过
        - name: format.directory_structure  # 仅目录模式激活，扁平文件模式自动跳过
          params:
            expected_modules: null          # 期望模块数量（null 表示不检查）
            hierarchy_depth: null           # 最大目录深度（null 表示不检查）
            expected_module_names: []       # 期望模块名称列表（空列表表示不检查）

    - id: commonsense
      name: "常识约束检查"
      short_circuit_policy: fail_fast
      evaluators:
        - name: commonsense.info_accuracy
          params:
            knowledge_base: "assets/knowledge/courseware/physics_v2.json"
        - name: commonsense.chronological_order
        - name: commonsense.logical_consistency
          params:
            llm_provider: deepseek_judge
        - name: commonsense.math_formula
        - name: commonsense.unit_consistency

    - id: quality
      name: "质量评估"
      short_circuit_policy: continue_all
      evaluators:
        # 软约束
        - name: soft.content_density
          params:
            min_chars_per_document: 200
            max_chars_per_document: 5000
          weight: 0.25
        - name: soft.visual_consistency
          weight: 0.25
        - name: soft.teaching_logic
          params:
            template_id: pedagogical_logic
          weight: 0.25
        - name: soft.content_diversity
          params:
            template_id: content_diversity
          weight: 0.25
        # 偏好约束
        - name: pref.style_preference
          params:
            template_id: style_preference
          weight: 0.33
        - name: pref.depth_preference
          params:
            template_id: depth_preference
          weight: 0.33
        - name: pref.request_fulfillment
          params:
            template_id: request_fulfillment
          weight: 0.34

  # 阈值设定 → 详见 06 评分与指标设计
  thresholds:
    DR: 0.95
    CPR: 0.90
    reward: 0.70
```

## 五、SUT 配置（sut_config.yaml）

```yaml
# 被测 Agent 连接配置
sut:
  name: "courseware-agent"
  driver: http
  base_url: "http://localhost:8000"
  timeout: 120

  # 请求模板
  request_template:
    method: POST
    path: "/generate"
    headers:
      Content-Type: "application/json"
      Authorization: "Bearer ${SUT_API_KEY}"
    body:
      subject: "{{task.input.subject}}"
      grade: "{{task.input.grade}}"
      topic: "{{task.input.topic}}"
      output_format: "html"              # 请求输出 HTML 文档集

  # 响应映射
  response_mapping:
    success_field: "success"
    output_files_field: "output_files"   # 返回的 MD/HTML 文件列表
    metadata_field: "metadata"
    error_field: "error"
```

> **注意**：当前阶段 Agent 产出物以手动放置为主。执行引擎（Sprint 8）就绪后将通过此配置自动获取 HTML 文档集。

## 五a、项目配置（project.yaml）

项目是评估运行的分组维度，用于 Web Portal 的项目看板与趋势追踪。

```yaml
# assets/projects/courseware-platform.yaml
project:
  id: "courseware-platform"                     # 项目唯一标识
  name: "课件平台 Agent"                         # 项目名称
  description: "课件生成 Agent 的评估项目"         # 项目描述
  default_rule_set: "courseware_v1"              # 默认使用的规则集
  default_task_set: "courseware_subject_coverage" # 默认使用的任务集
```

**使用方式**：

```bash
# 评估时指定项目
agent-eval eval --package-dir ./packages/run_001/ --rule-set ./rules/courseware_v1.yaml --project courseware-platform

# Web Portal 按项目分组展示趋势
agent-eval serve
```

**项目与运行的关系**：

```
project.yaml                运行时关联
┌──────────────────┐       ┌─────────────────────────┐
│ courseware-      │──→    │ run_20260608_143000      │
│ platform         │       │ run_20260607_100000      │
│                  │       │ run_20260606_090000      │
└──────────────────┘       └─────────────────────────┘
                                  ↓
                           projects.json + runs_index.json
                           （Web Portal 读取）
```

当运行时通过 `--project` 参数指定项目 ID 时，系统自动更新 `workspace/index/` 下的索引文件。若未指定项目，运行结果归入默认项目 `"default"`。

## 九、Markdown 评测计划规范（eval_plan.md）

### 9.1 概述

Markdown 评测计划（eval_plan.md）是系统的**核心配置入口**，采用 YAML frontmatter + Markdown 正文的混合格式，兼顾机器可解析性与人类可读性。它取代了 `pipeline.yaml` 中的级联阶段定义逻辑，将评测背景、策略、阈值与预算统一管理。

**存放位置**：`assets/plans/*.md`

**设计动机**：
- 传统 `pipeline.yaml` 仅描述阶段和评估器的机械组合，缺乏评测背景与策略意图的表达
- Markdown 格式允许直接作为 Agent 的 system prompt 片段注入，实现"评测即引导"
- frontmatter 提供结构化元数据，正文提供自然语言策略描述，两者协同工作

### 9.2 文件格式

```
---
# YAML frontmatter（结构化元数据）
plan_id: <string>
version: <string>
target: <string>
max_turns: <integer>
budget_usd: <float>
---

# Markdown 正文（自然语言策略描述）

## 评测背景
...

## 评测策略
...

## 阈值
...

## 预算
...
```

### 9.3 Frontmatter 字段定义

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `plan_id` | string | 是 | 评测计划唯一标识，如 `courseware-eval-v2` |
| `version` | string | 是 | 计划版本号，语义化版本格式，如 `"2.0.0"` |
| `target` | string | 是 | 评测目标描述，如 `"courseware-generation"` |
| `max_turns` | integer | 是 | Agent 单次评测最大对话轮次 |
| `budget_usd` | float | 是 | 单次评测预算上限（美元） |

### 9.4 正文结构

Markdown 正文必须包含以下一级标题章节：

#### 评测背景

描述本次评测的背景信息，包括：
- 被测 Agent 的定位与能力范围
- 评测目标与关注点
- 适用的场景范围

#### 评测策略

描述各评测阶段的策略，按级联顺序组织为二级标题子章节。每个子章节对应一个评测阶段，包含：
- 阶段目标
- 使用的评估器及其参数
- 通过/失败判定逻辑
- 失败后的处理策略（短路终止 / 继续执行）

典型子章节包括：
- **格式约束检查**：对应 `format` 阶段
- **常识约束检查**：对应 `commonsense` 阶段
- **质量评估**：对应 `quality` 阶段

#### 阈值

定义各级评测指标的通过阈值，通常包括：
- DR（交付率）阈值
- CPR（约束通过率）阈值
- Reward（综合得分）阈值

#### 预算

定义评测执行的预算控制策略，包括：
- 单次评测预算上限
- LLM 调用预算分配
- 超预算处理策略

### 9.5 完整示例：课件评测计划

```markdown
---
plan_id: courseware-eval-v2
version: "2.0.0"
target: courseware-generation
max_turns: 30
budget_usd: 1.50
---

# 课件生成 Agent 评测计划

## 评测背景

本计划用于评测课件生成 Agent 的整体输出质量。被测 Agent 接收学科、年级、知识点等输入参数，
生成 Markdown/HTML 格式的课件文档集。评测覆盖格式合规性、内容准确性、教学适用性三个层次，
采用级联评估策略：格式不合规则短路终止，常识错误记录但继续执行，质量评估全部完成后汇总。

适用场景：K12 课件生成、大单元教学资料生成。

## 评测策略

### 格式约束检查

**阶段目标**：验证 Agent 输出的文档集在格式层面满足基本要求。

**评估器配置**：

| 评估器 | 关键参数 | 说明 |
|--------|----------|------|
| `format.response_format` | `allowed_formats: ["md", "html"]` | 检查输出文件格式是否为允许的类型 |
| `format.document_count` | `min_documents: 1, max_documents: 20` | 检查文档数量是否在合理范围 |
| `format.structure_compliance` | `max_heading_depth: 4` | 检查标题层级合规性 |
| `format.html_validity` | `check_html_only: true` | HTML 文件标签合法性检查 |
| `format.directory_structure` | `expected_modules: null, hierarchy_depth: null` | 目录模式下的结构检查 |

**判定策略**：fail_fast —— 任一评估器失败则短路终止后续阶段。

### 常识约束检查

**阶段目标**：验证课件内容不存在事实性错误、时间线混乱、逻辑矛盾等常识性问题。

**评估器配置**：

| 评估器 | 关键参数 | 说明 |
|--------|----------|------|
| `commonsense.info_accuracy` | `knowledge_base: "assets/knowledge/courseware/physics_v2.json"` | 事实准确性校验 |
| `commonsense.chronological_order` | — | 时间线顺序校验 |
| `commonsense.logical_consistency` | `llm_provider: deepseek_judge` | 逻辑一致性校验（LLM 辅助） |
| `commonsense.math_formula` | — | 数学公式正确性校验 |
| `commonsense.unit_consistency` | — | 单位一致性校验 |

**判定策略**：fail_fast —— 发现严重常识错误即终止。

### 质量评估

**阶段目标**：对课件的教学适用性、内容深度、风格偏好等进行综合评分。

**软约束评估器（权重合计 1.0）**：

| 评估器 | 权重 | 说明 |
|--------|------|------|
| `soft.content_density` | 0.25 | 内容密度（每文档 200-5000 字） |
| `soft.visual_consistency` | 0.25 | 视觉风格一致性 |
| `soft.teaching_logic` | 0.25 | 教学逻辑性（使用 pedagogical_logic 模板） |
| `soft.content_diversity` | 0.25 | 内容多样性 |

**偏好约束评估器（权重合计 1.0）**：

| 评估器 | 权重 | 说明 |
|--------|------|------|
| `pref.style_preference` | 0.33 | 风格偏好匹配度 |
| `pref.depth_preference` | 0.33 | 深度偏好匹配度 |
| `pref.request_fulfillment` | 0.34 | 需求满足度 |

**判定策略**：continue_all —— 所有评估器并行执行，汇总计算 Reward 得分。

## 阈值

| 指标 | 阈值 | 说明 |
|------|------|------|
| DR（交付率） | 0.95 | 格式+常识约束通过率不低于 95% |
| CPR（约束通过率） | 0.90 | 全部约束（含软约束+偏好）通过率不低于 90% |
| Reward（综合得分） | 0.70 | 加权综合得分不低于 0.70 |

## 预算

| 项目 | 配置 | 说明 |
|------|------|------|
| 单次评测上限 | $1.50 | 包含所有 LLM 调用 |
| LLM Judge 调用 | ≤ $1.00 | 常识检查 + 质量评估的 LLM 调用 |
| 预算超限策略 | 降级评估 | 超预算时跳过 LLM 评估器，仅执行规则评估器 |
```

### 9.6 与 Agent System Prompt 的映射

评测计划通过 `EvalPlan.to_system_prompt()` 方法转换为 Agent 的系统提示片段：

```python
class EvalPlan:
    """评测计划数据模型"""

    plan_id: str
    version: str
    target: str
    max_turns: int
    budget_usd: float
    body: str  # Markdown 正文原文

    def to_system_prompt(self) -> str:
        """将评测计划转换为 Agent system prompt 片段

        返回内容包含：
        1. frontmatter 中的结构化参数（max_turns、budget 等作为执行约束）
        2. Markdown 正文作为策略指导（评测背景、策略、阈值）
        """
        header = (
            f"## 评测计划: {self.plan_id} (v{self.version})\n"
            f"- 评测目标: {self.target}\n"
            f"- 最大轮次: {self.max_turns}\n"
            f"- 预算上限: ${self.budget_usd:.2f}\n\n"
        )
        return header + self.body
```

**注入位置**：`to_system_prompt()` 的输出作为 Agent 系统提示的一部分，位于"角色定义"之后、"执行指令"之前，使 Agent 在执行评测时能够理解评测策略的全局上下文。

### 9.7 与其他配置文件的关系

```
eval_plan.md                 关系说明
┌─────────────────────┐
│ YAML frontmatter    │──── max_turns, budget_usd ────→ Agent 运行时约束
│ (plan_id, version,  │
│  target, ...)       │
├─────────────────────┤
│ Markdown 正文       │
│ ┌─────────────────┐ │
│ │ 评测策略         │──── 替代 ────→ pipeline.yaml 的 stages 级联逻辑
│ │ (阶段/评估器/    │     （eval_plan.md 优先级更高，
│ │  判定策略)       │      pipeline.yaml 作为兼容回退）
│ └─────────────────┘ │
│ ┌─────────────────┐ │
│ │ 阈值/预算        │──── 替代 ────→ pipeline.yaml 的 thresholds 字段
│ └─────────────────┘ │     （统一纳入计划文件管理）
└─────────────────────┘
         │
         │ 互补（不替代）
         ↓
┌─────────────────────┐
│ rule_set.yaml       │  具体规则定义（规则ID、维度、惩罚）
│ task_set.yaml       │  具体任务数据（输入、约束、期望）
│ prompt 模板          │  LLM Judge 评估模板
│ llm_config.yaml     │  LLM Provider 连接配置
└─────────────────────┘
```

**核心关系**：
- **替代 pipeline.yaml 的级联逻辑**：eval_plan.md 中的"评测策略"章节描述了阶段、评估器、判定策略，取代了 pipeline.yaml 中 `stages` 字段的机械定义。pipeline.yaml 仍可作为兼容回退，当 eval_plan.md 不存在时使用。
- **互补 rule_set.yaml**：eval_plan.md 描述"评什么、怎么评"的策略层面，rule_set.yaml 定义具体的规则 ID、维度、惩罚等数据层面。两者协同工作。
- **互补 task_set.yaml**：eval_plan.md 不包含具体的任务输入数据，任务定义仍在 task_set.yaml 中管理。
- **互补 prompt 模板**：eval_plan.md 引用评估器名称（如 `soft.teaching_logic`），评估器对应的 LLM Prompt 模板仍在 `assets/prompts/` 中独立管理。

## 十、Agent 模式配置

### 10.1 概述

Agent 模式是系统的高级运行模式，通过配置驱动 Agent 自主完成评测任务。本节定义 Agent 模式相关的配置数据模型与集成方式。

### 10.2 核心配置模型

#### AgentConfig

Agent 全局配置，定义 Agent 运行环境的基本参数：

```python
class AgentConfig:
    """Agent 全局配置"""

    # 使用的 LLM 模型标识
    model: str = "claude-sonnet-4-20250514"

    # Agent 工作空间目录（存放评测输入/输出）
    workspace_dir: str = "./workspace"

    # 日志输出目录
    log_dir: str = "./workspace/logs"
```

**字段说明**：

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `model` | string | `"claude-sonnet-4-20250514"` | Agent 使用的 LLM 模型标识，支持 Claude、DeepSeek 等 |
| `workspace_dir` | string | `"./workspace"` | 工作空间根目录，存放评测包、结果和索引 |
| `log_dir` | string | `"./workspace/logs"` | Agent 运行日志目录 |

#### AgentSUTConfig

被测 Agent（SUT）的交互配置，定义系统如何与被测 Agent 通信：

```python
class AgentSUTConfig:
    """被测 Agent (SUT) 交互配置"""

    # 系统提示模板（支持引用 eval_plan.md 的内容）
    system_prompt_template: str = ""

    # Agent 允许使用的工具列表
    allowed_tools: list[str] = field(default_factory=lambda: [
        "file_read",
        "file_write",
        "file_list",
        "http_request",
    ])

    # 单次评测最大对话轮次（可被 eval_plan.md 中的 max_turns 覆盖）
    max_turns: int = 30

    # 被测 Agent 的工作目录
    working_dir: str = "./workspace/sut_output"
```

**字段说明**：

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `system_prompt_template` | string | `""` | 系统提示模板，支持 `{plan_body}` 等占位符，运行时替换为 eval_plan.md 的正文内容 |
| `allowed_tools` | list[str] | `["file_read", "file_write", "file_list", "http_request"]` | Agent 可调用的工具白名单，控制 Agent 的能力边界 |
| `max_turns` | int | `30` | 最大对话轮次，eval_plan.md 中定义的 `max_turns` 优先级更高 |
| `working_dir` | string | `"./workspace/sut_output"` | 被测 Agent 的输出工作目录 |

### 10.3 CLI 集成

Agent 模式通过 CLI 命令行参数启用和配置：

```bash
# 基本用法：指定评测计划启动 Agent 模式
agent-eval agent --plan assets/plans/courseware-eval-v2.md

# 完整参数示例
agent-eval agent \
  --plan assets/plans/courseware-eval-v2.md \
  --max-turns 50 \
  --budget-usd 2.00 \
  --agent-model claude-sonnet-4-20250514 \
  --workspace-dir ./workspace \
  --log-dir ./workspace/logs \
  --rule-set ./rules/courseware_v1.yaml \
  --task-set ./tasks/courseware/subject_coverage.yaml
```

**CLI 参数说明**：

| 参数 | 类型 | 说明 | 优先级 |
|------|------|------|--------|
| `--plan` | path | 评测计划文件路径（eval_plan.md），启用 Agent 模式的必要参数 | — |
| `--max-turns` | int | 覆盖 frontmatter 中的 `max_turns` | CLI > eval_plan.md > 默认值 |
| `--budget-usd` | float | 覆盖 frontmatter 中的 `budget_usd` | CLI > eval_plan.md > 默认值 |
| `--agent-model` | string | 指定 Agent 使用的 LLM 模型，覆盖 `AgentConfig.model` | CLI > 配置文件 > 默认值 |
| `--workspace-dir` | path | 覆盖 `AgentConfig.workspace_dir` | CLI > 配置文件 |
| `--log-dir` | path | 覆盖 `AgentConfig.log_dir` | CLI > 配置文件 |
| `--rule-set` | path | 使用的规则集文件（互补配置） | — |
| `--task-set` | path | 使用的任务集文件（互补配置） | — |

**优先级规则**：CLI 参数 > eval_plan.md frontmatter > 配置文件默认值。

### 10.4 SDK 集成示例

通过 Python SDK 以编程方式使用 Agent 模式：

```python
from agent_eval.agent import AgentEvaluator
from agent_eval.config import AgentConfig, AgentSUTConfig

# 1. 加载评测计划
plan = AgentEvaluator.load_plan("assets/plans/courseware-eval-v2.md")

# 2. 配置 Agent
agent_config = AgentConfig(
    model="claude-sonnet-4-20250514",
    workspace_dir="./workspace",
    log_dir="./workspace/logs",
)

sut_config = AgentSUTConfig(
    system_prompt_template="你是一个课件生成评估专家。\n\n{plan_body}",
    allowed_tools=["file_read", "file_write", "file_list", "http_request"],
    max_turns=plan.max_turns,  # 使用计划中的轮次限制
    working_dir="./workspace/sut_output",
)

# 3. 创建评估器并执行
evaluator = AgentEvaluator(
    plan=plan,
    agent_config=agent_config,
    sut_config=sut_config,
    rule_set_path="assets/rules/courseware/format_constraints.yaml",
    task_set_path="assets/tasks/courseware/subject_coverage.yaml",
)

# 4. 运行评测
result = evaluator.evaluate(
    task_id="math_grade7_001",
    budget_usd=1.50,  # 可在运行时覆盖预算
)

# 5. 输出结果
print(f"评测结果: {'通过' if result.passed else '未通过'}")
print(f"DR: {result.metrics.dr:.2%}")
print(f"CPR: {result.metrics.cpr:.2%}")
print(f"Reward: {result.metrics.reward:.2f}")
print(f"实际花费: ${result.cost_usd:.4f}")
```
