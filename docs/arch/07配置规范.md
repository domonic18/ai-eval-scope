# 配置规范

> 本文档定义系统所有配置文件的格式与 Schema，属于 [01 整体架构设计](./01整体架构设计.md) §二中"基础设施层"的详细展开。评估侧数据模型参见 [04 评估引擎设计](./04评估引擎设计.md)，执行侧数据模型参见 [03 执行引擎设计](./03执行引擎设计.md)。

---

## 一、配置文件总览

| 配置文件 | 存放位置 | 作用 |
|----------|----------|------|
| `pipeline.yaml` | `assets/rules/` | 管线配置：阶段、评估器、权重、阈值 |
| `rule_set.yaml` | `assets/rules/{scenario}/` | 规则集：具体规则定义 |
| `task_set.yaml` | `assets/tasks/{scenario}/` | 任务集：输入数据定义 |
| `project.yaml` | `assets/projects/` | 项目配置：评估运行分组 |
| `sut_config.yaml` | CLI 参数或 `assets/` | 被测 Agent 连接配置 |
| `llm_config.yaml` | CLI 参数或 `assets/` | LLM Provider 配置 |
| Prompt 模板 | `assets/prompts/` | LLM Judge 的评估模板 |
| JSON Schema | `assets/schemas/` | 配置文件校验 Schema |

## 二、管线配置（pipeline.yaml）

```yaml
# assets/rules/_default.yaml
pipeline:
  name: courseware-eval-v1
  version: "1.0"
  description: "课件生成Agent标准评估管线"

  # Reward 公式参数 → 详见 06 评分与指标设计
  reward_weights:
    w3: 1.0    # 软约束乘数
    w4: 1.0    # 偏好约束乘数

  # 级联阶段定义 → 详见 03 管线引擎设计
  stages:
    - id: format
      name: "格式约束检查"
      short_circuit_policy: fail_fast
      evaluators:
        - name: format.response_format
          params:
            allowed_formats: ["md", "html"]
        - name: format.document_count
          params:
            min_documents: 1
            max_documents: 20
        - name: format.structure_compliance
          params:
            max_heading_depth: 4          # 标题最大层级
            require_toc: false            # 是否要求目录
        - name: format.html_validity
          params:
            check_html_only: true         # 仅检查 HTML 文件，Markdown 跳过
        - name: format.directory_structure  # 仅目录模式激活，扁平文件模式自动跳过
          params:
            expected_modules: null          # 期望模块数量（null 表示不检查）
            hierarchy_depth: null           # 最大目录深度（null 表示不检查）
            expected_module_names: []       # 期望模块名称列表（空列表表示不检查）

    - id: commonsense
      name: "常识约束检查"
      short_circuit_policy: fail_fast
      evaluators:
        - name: commonsense.info_accuracy
          params:
            knowledge_base: "assets/knowledge/courseware/physics_v2.json"
        - name: commonsense.chronological_order
        - name: commonsense.logical_consistency
          params:
            llm_provider: deepseek_judge
        - name: commonsense.math_formula
        - name: commonsense.unit_consistency

    - id: quality
      name: "质量评估"
      short_circuit_policy: continue_all
      evaluators:
        # 软约束
        - name: soft.content_density
          params:
            min_chars_per_document: 200
            max_chars_per_document: 5000
          weight: 0.25
        - name: soft.visual_consistency
          weight: 0.25
        - name: soft.teaching_logic
          params:
            template_id: pedagogical_logic
          weight: 0.25
        - name: soft.content_diversity
          params:
            template_id: content_diversity
          weight: 0.25
        # 偏好约束
        - name: pref.style_preference
          params:
            template_id: style_preference
          weight: 0.33
        - name: pref.depth_preference
          params:
            template_id: depth_preference
          weight: 0.33
        - name: pref.request_fulfillment
          params:
            template_id: request_fulfillment
          weight: 0.34

  # 阈值设定 → 详见 06 评分与指标设计
  thresholds:
    DR: 0.95
    CPR: 0.90
    reward: 0.70
```

## 三、规则集配置（rule_set.yaml）

```yaml
# assets/rules/courseware/format_constraints.yaml
version: "1.0"
description: "课件生成评估规则集"
schema: "./assets/schemas/rule_set_schema.json"

dimensions:
  - id: functional
    name: "功能性"
    weight: 1.0
  - id: effectiveness
    name: "效果性"
    weight: 1.0
  - id: safety
    name: "安全性"
    weight: 1.0
  - id: performance
    name: "性能"
    weight: 0.5

cascade:
  - stage: format_gate
    name: "格式门控"
    stop_on_fail: true
  - stage: commonsense_gate
    name: "常识门控"
    stop_on_fail: false
  - stage: quality_eval
    name: "质量评估"
    stop_on_fail: false

rules:
  - id: FMT_001
    name: "输出格式有效"
    dimension: functional
    stage: format_gate
    description: "SUT 必须生成有效的 Markdown 或 HTML 文件"
    evaluator: format.response_format
    params:
      allowed_formats: ["md", "html"]
    weight: 1.0
    penalty_on_fail: -3

  - id: FMT_002
    name: "文档数量合规"
    dimension: functional
    stage: format_gate
    description: "输出文档数量应在任务要求的范围内"
    evaluator: format.document_count
    params:
      min: "{{task.constraints.min_documents | default(1)}}"
      max: "{{task.constraints.max_documents | default(20)}}"
    weight: 1.0
    penalty_on_fail: -3

  - id: FMT_005
    name: "目录结构合规"
    dimension: functional
    stage: format_gate
    description: "目录模式下，目录层级、模块数量和名称应符合预期"
    evaluator: format.directory_structure
    params:
      expected_modules: "{{task.constraints.expected_modules | default(null)}}"
      hierarchy_depth: "{{task.constraints.hierarchy_depth | default(null)}}"
      expected_module_names: "{{task.constraints.expected_module_names | default([])}}"
    weight: 1.0
    penalty_on_fail: -3

  # 更多规则项...
```

      grade: 3
      topic: "大单元学习总导"

      min_documents: 20                 # 最少文件数
      max_documents: 30                 # 最多文件数
      format: ["html"]                  # 期望的文件格式
      hierarchy_depth: 3                # 期望的目录深度
      expected_modules: 2               # 期望的顶层模块数
    constraints:
      min_documents: 20                 # 最少文件数
      max_documents: 30                 # 最多文件数
      format: ["html"]                  # 期望的文件格式
      hierarchy_depth: 3                # 期望的目录深度
      expected_modules: 2               # 期望的顶层模块数
      expected_module_names:            # 期望的模块名称（用于 directory_structure 评估器）

    collector_options:                  # DirectoryCollector 运行时参数

## 五、SUT 配置（sut_config.yaml）

```yaml
# 被测 Agent 连接配置
sut:
  name: "courseware-agent"
  driver: http
  base_url: "http://localhost:8000"
  timeout: 120

  # 请求模板
  request_template:
    method: POST
    path: "/generate"
    headers:
      Content-Type: "application/json"
      Authorization: "Bearer ${SUT_API_KEY}"
    body:
      subject: "{{task.input.subject}}"
      grade: "{{task.input.grade}}"
      topic: "{{task.input.topic}}"
      output_format: "html"              # 请求输出 HTML 文档集

  # 响应映射
  response_mapping:
    success_field: "success"
    output_files_field: "output_files"   # 返回的 MD/HTML 文件列表
    metadata_field: "metadata"
    error_field: "error"
```

> **注意**：当前阶段 Agent 产出物以手动放置为主。执行引擎（Sprint 8）就绪后将通过此配置自动获取 HTML 文档集。

## 五a、项目配置（project.yaml）

项目是评估运行的分组维度，用于 Web Portal 的项目看板与趋势追踪。

```yaml
# assets/projects/courseware-platform.yaml
project:
  id: "courseware-platform"                     # 项目唯一标识
  name: "课件平台 Agent"                         # 项目名称
  description: "课件生成 Agent 的评估项目"         # 项目描述
  default_rule_set: "courseware_v1"              # 默认使用的规则集
  default_task_set: "courseware_subject_coverage" # 默认使用的任务集
```

**使用方式**：

```bash
# 评估时指定项目
agent-eval eval --package-dir ./packages/run_001/ --rule-set ./rules/courseware_v1.yaml --project courseware-platform

# Web Portal 按项目分组展示趋势
agent-eval serve
```

**项目与运行的关系**：

```
project.yaml                运行时关联
┌──────────────────┐       ┌─────────────────────────┐
│ courseware-      │──→    │ run_20260608_143000      │
│ platform         │       │ run_20260607_100000      │
│                  │       │ run_20260606_090000      │
└──────────────────┘       └─────────────────────────┘
                                  ↓
                           projects.json + runs_index.json
                           （Web Portal 读取）
```

当运行时通过 `--project` 参数指定项目 ID 时，系统自动更新 `workspace/index/` 下的索引文件。若未指定项目，运行结果归入默认项目 `"default"`。

## 八、JSON Schema 校验

系统提供 JSON Schema 用于校验配置文件的合法性：

| Schema 文件 | 校验对象 |
|-------------|----------|
| `assets/schemas/rule_set_schema.json` | rule_set.yaml |
| `assets/schemas/task_set_schema.json` | task_set.yaml（含目录模式字段） |
| `assets/schemas/sut_config_schema.json` | sut_config.yaml |
| `assets/schemas/project_schema.json` | project.yaml |

Schema 校验在配置加载时自动执行，支持：
- IDE 通过 YAML Schema 关联实时提示
- Agent 编辑后程序化校验
- CLI 启动时前置检查
