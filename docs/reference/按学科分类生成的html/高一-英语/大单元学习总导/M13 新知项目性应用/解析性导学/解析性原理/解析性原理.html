🧩 解析性原理

从具体案例中提炼可迁移的分析方法、判断规则和思维路径

💡 为什么要提炼原理？案例是具体的、一次性的，但原理是可迁移的、反复可用的。学会从具体案例中抽象出通用方法，才是真正的学习。本节承接解析性示例中的分析示范，帮你提炼出通用原理。

📐 解析性原理提炼

点击每条原理查看详细说明、示例体现、使用方法和迁移场景。

🔍 原理1：问题定位法（Find the Core）

▼
📌 说明 在友谊冲突中，先定位"真正的问题是什么"，而不是停留在表面行为上。
本示例中的体现 示例中先问"他们之间真正的矛盾是什么"而非仅看"吵架了"。
如何使用 面对一段友谊问题，先问 3 个问题——What happened? How do they feel? What do they really want?
可迁移到 所有需要分析人际关系问题的场景（家庭矛盾、小组合作冲突等）
⚠️ 使用提醒 不要过早下结论，先收集足够的信息。

🔄 原理2：视角切换法（Perspective Switch）

▼
📌 说明 站在双方的角度分别分析，理解各自的立场和感受。
本示例中的体现 示例中分别列出了小明视角和 David 视角的感受。
如何使用 用 T-chart 分别写出双方的感受、需求和顾虑。
可迁移到 任何涉及多方利益的矛盾分析
⚠️ 使用提醒 视角切换 ≠ 否定自己的感受，而是先理解再沟通。

🧱 原理3：结构化分析法（Structure It）

▼
📌 说明 把模糊的问题拆解为结构化的要素（问题 → 原因 → 影响 → 解决方案）。
本示例中的体现 按照"事实 → 感受 → 方案 → 期望"的结构分析。
如何使用 用"问题分析表"逐项填写。
可迁移到 议论文写作、问题解决类任务
⚠️ 使用提醒 结构是工具不是枷锁，灵活调整。

📎 原理4：证据支撑法（Use Evidence）

▼
📌 说明 分析时引用具体事实和细节，而不是笼统地判断。
本示例中的体现 示例用具体事例（"在准备展示时，David 没有等小明就决定了分工"）支撑判断。
如何使用 在写下判断前先问 "What is my evidence?"
可迁移到 议论文写作、演讲、辩论
⚠️ 使用提醒 证据要具体、可验证。

✏️ 解析性原理应用练习

练习1：分析一次小组合作中的矛盾

情境：你参加了一个英语小组展示，组员 A 觉得组员 B 不认真，组员 B 觉得组员 A 太专制。

解析任务：用 问题定位法 + 视角切换法 分析这个矛盾。

建议原理：原理1 + 原理2

分析提示：不要判断谁对谁错，先列出双方可能看到的"事实"。

🧰 脚手架提示：用 T-chart 列出双方视角

练习2：分析一段课文中的友谊关系

情境：阅读教材中的 "Friendship in Hawaii" 篇章。

解析任务：用 结构化分析法 分析夏威夷文化中的友谊观。

建议原理：原理3 + 原理4

分析提示：注意作者用了哪些具体例子来支撑关于"Aloha 精神"的描述。

🧰 脚手架提示：找出文中的 3 个关键细节
