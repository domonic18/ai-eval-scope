🧪 从"厨房里的化学"到"系统性思维"

一个小苏打探究的建构之旅 · 九年级化学

📖 真实情境

周末你在厨房帮妈妈做馒头，看到妈妈往面粉里加了一小包白色粉末。你好奇地问这是什么，妈妈说这是"小苏打"。你查了一下手机，发现小苏打（碳酸氢钠）不仅用于做馒头，还用于清洁、灭火……

你突然产生了一个疑问：一种物质为什么能有这么多不同的用途？这里面隐藏着怎样的化学奥秘？

建构目标：建构一个"物质性质—用途关系系统模型"，解释小苏打为什么能用于膨松剂、清洁剂和灭火等不同场景

🎯 建构目标与任务

成果名称：物质性质—用途关联系统模型（手绘图 + 说明）

最低标准：至少包含 3种用途 及其对应的 性质解释，展现 因果关系 和系统关联

关键句式：

"因为物质具有 ______性质（证据：______），所以它可以被用于 ______"

📋 建构过程（6 步）👇 点击展开详情

1提出问题，收集证据
确定探究对象，查阅资料，制作"小苏打信息档案卡"

做什么：确定探究对象（小苏打），列出已知信息和疑问

怎么做：查阅教材和可靠资料，收集小苏打的物理性质、化学性质

生成：一份"小苏打信息档案卡"，包括外观、溶解性、热稳定性、酸碱性等

2发现多用途，建立关联
列出小苏打的3—5种日常用途，标注对应性质

做什么：列出小苏打的3—5种日常用途

怎么做：从生活经验中回忆（做馒头、清洁、除臭、灭火等），查阅资料补充

生成：一张"用途清单"，每个用途旁标注猜测对应的性质

3验证猜想，寻找证据
设计简单验证，确认用途与性质的对应关系

做什么：设计简单验证，确认用途与性质的对应关系

怎么做：小苏打遇醋冒泡→产生CO₂→解释膨松和灭火原理；小苏打水溶液微碱性→解释清洁除油原理

生成：一份"用途—性质证据对照表"

4系统建模，构建关系网络
将单个"用途↔性质"关系连接成系统网络

做什么：将单个的"用途↔性质"关系连接成系统网络

怎么做：用概念图/思维导图形式，将小苏打的分子结构、物理性质、化学性质、各种用途用箭头联系起来，标注因果关系

生成：第一版"物质性质—用途系统模型草图"

5迭代优化，补充细节
检查模型是否有遗漏或错误，补充细节

做什么：检查模型是否有遗漏或错误

怎么做：对照教材和资料，补充如"分解产生CO₂的具体化学方程式""碱性清洁的作用机制"等细节

生成：第二版完整模型，用颜色区分已知和新增信息

6成果表达，清晰呈现
将完整模型以适合展示的形式呈现

做什么：将完整的系统模型以适合展示的形式呈现

怎么做：使用双色笔，在A4纸上绘制最终版概念图，配上简要文字说明

生成：完整的"物质性质—用途关联系统模型"成品

📊 建构成果样例

🧬 碳酸氢钠（小苏打）性质—用途系统模型

评价要求：

  • 科学性：所有性质描述准确，用途与性质的对应合理
  • 完整性：覆盖物理性质（颜色、状态、溶解性）和化学性质（热稳定性、酸碱性）
  • 系统性：至少包含3个用途节点，用箭头标明因果关系

推荐格式：A4纸横用，中心图+辐射分支，蓝色标性质、红色标用途、绿色箭头标因果关系

🚀 优化建议

  1. 查漏补缺：对照教材中关于物质性质与用途的章节，检查模型是否涵盖了所有关键性质
  2. 升级模型：尝试加入"物质结构→性质→用途"完整链条
  3. 跨界迁移：用同样的方法去分析其他物质（如食盐、食醋）
