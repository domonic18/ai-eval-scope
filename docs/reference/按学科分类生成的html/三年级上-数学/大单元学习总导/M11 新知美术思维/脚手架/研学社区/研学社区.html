身边的设计

🏪 超市的营业时间表设计

背景：你家附近的超市营业时间是8:00-22:00，超市入口处有一张"各区域营业时间表"
生鲜区6:00就开门（早于超市开门），药品区24小时开放，其他区域8:00-22:00。怎么用一张图把"不同区域不同营业时间"说清楚？
① 用时间轴（每行为一个区域）② 用颜色区分区域 ③ 用色块标注营业时间段
大多数超市选择了"纵向时间轴 + 横向区域排列"的表格形式
当你要表达"多个事物各有各的时间段"时，可以用"一个事物一行"的时间轴排列法

"如果你是决策者"任务

如果你要重新设计这张超市营业时间表，使它更清晰美观，你会怎么改？画出你的改进方案。

设计师小提示：

  • 可以用不同颜色区分区域（生鲜🟢、药品🔵、日用🟡）
  • 可以在表格顶部标注当前时间的位置
  • 可以用图标代替文字，更直观

进阶层 · 行业 / 历史案例

专业设计

🏅 奥运会的赛程表设计

背景：奥运会要在16天内举办数百场比赛，观众需要知道"什么比赛在什么时间什么场馆举行"
怎么把几百场比赛的信息组织成观众"一眼就能查到自己想看比赛"的赛程表？
① 按日期分组 ② 按项目分类 ③ 用不同颜色区分不同大项（田径=蓝、游泳=绿、体操=红……）
形成了"日赛程表" + "项目总表"的双表结构，配以颜色编码
信息量大时，先分组再分层，颜色编码是快速定位的关键

"如果你是决策者"任务

假如你是2028年奥运会赛程表的设计师，你要在手机小屏幕上展示赛程——和纸质的赛程表设计有什么不同？你会去掉什么？保留什么？

手机屏幕设计要点：

  • 去掉：所有非核心信息（场馆座位图、运动员介绍等）
  • 保留：比赛项目 + 时间 + 场馆 + 是否决赛（⭐标注）
  • 新加：上下滑动代替翻页，点击项目查看详情
  • 颜色：用更鲜艳的色块区分项目大项

挑战层 · 复杂系统案例

系统思维

🚌 城市公交调度"时间-车辆"可视化

背景：一座中型城市有50条公交线路、300辆公交车，每天运营时间6:00-22:00
调度员需要实时知道"每辆车现在在哪条线上、要不要发下一班"
① 每条线路画一条时间轴 ② 每趟车用一条斜线表示 ③ 斜线的位置和角度反映出发时间和到站时间
形成了"巴士运行图"——看起来像地铁图，但实际上每条线的角度代表了时间
最复杂的时间可视化往往需要"时间 + 空间"的二维表达
