1
个人理解型
用身体感知时间
▼
真实建构情境
上课时老师突然说："接下来我们安静地坐1分钟，不准看钟表，你觉得到了就举手。"结果有的同学20秒就举手了，有的过了2分钟还没举。你发现自己根本不知道"1分钟"到底有多长！你决定用身体做一个"内置计时器"。
建构任务
设计一份"我的身体计时器"记录表，找到至少3种用身体感知1分钟的方法。
原理调用提示
用"情境绑定法"——把1分钟和身体动作绑在一起（比如默念一遍《静夜思》、闭眼深呼吸15次）。再用"多感官验证法"——看手表核对你的身体估算准不准。
建构步骤
  1. 闭上眼睛感受1分钟——在心里数"1、2、3……"，数到60就睁眼，看看过了多久。记录偏差。
  2. 换一种方式：闭上眼、深呼吸15次，睁眼看表。记录偏差。
  3. 再换一种：在心里默唱一首你最喜欢的歌的一段（比如生日歌唱两遍），睁眼看表。记录偏差。
  4. 对比3次记录，找出你最准的"身体计时法"。
  5. 把结果画成一张"我的身体计时器"记录表。
脚手架推荐
秒表（手机或手表的计时功能）、一张空白A4纸折成4格分别记录3种方法+结论、彩色笔标注偏差。
成果表达
"我的身体计时器"记录表——A4纸，4个区域：方法一/方法二/方法三/我的最佳身体计时法。每种方法写：方法名称、我感受到的时长（秒）、实际时长（秒）、偏差（秒）。
自评量规
  • [ ] 我至少尝试了3种不同的身体计时方法
  • [ ] 我的记录表上有每种方法的"感受时长"和"实际时长"
  • [ ] 我计算了每种方法的偏差（偏差=实际−感受）
  • [ ] 我选出了自己最准的一种身体计时法并标注了原因
  • [ ] 记录表整洁、字迹清晰，能给别人看懂
2
生活应用型
安排周末时间
▼
真实建构情境
周末到了，你想做的事一大堆：看动画片、搭乐高、下楼踢球、读课外书、还要写作业。妈妈只说了一句"自己安排时间"。你发现如果不会分时间，一整天可能啥也干不成。
建构任务
用"时间块"规划一个周末半天的方案，精确到30分钟。
原理调用提示
用"刻度拆解法"——把半天（比如6小时）拆成12个30分钟的时间块。用"情境绑定法"——给每个时间块绑定一个具体活动。
建构步骤
  1. 列出这半天你想做的所有事，估算每件事需要的时间。
  2. 把半天（如8:00~12:00）画成一条时间轴，分成30分钟一格。
  3. 把每件事填入对应的时间格，注意留出休息和缓冲时间。
  4. 检查：所有事加起来的时间≦半天总时间吗？超了就去掉不重要的。
  5. 按你的规划执行一次，执行后在旁边标注实际用时。
脚手架推荐
时间轴模板（画一条横线，每2厘米标一个30分钟格）、彩色便签纸或彩笔、计时器。
成果表达
周末时间分配方案——一张时间轴图，标注每个时间段的安排。完成后附上"实际执行记录"：计划和实际分别用了多少时间。
自评量规
  • [ ] 我的时间轴覆盖了完整的半天（至少4小时）
  • [ ] 我把半天分成了至少6个时间块
  • [ ] 每个时间块标注了活动内容和时长
  • [ ] 所有时间块加起来不超过半天总时间
  • [ ] 我实际执行了规划并记录了偏差
3
校园问题解决型
课间10分钟能做什么
▼
真实建构情境
下课铃响了！你冲出去玩耍，结果玩得太疯，上课铃响时你才想起来：厕所还没上、下节课的书还没拿出来、口渴得不行……你决定设计一份"完美课间方案"。
建构任务
设计一张课间10分钟"时间拼图"，把必须做的事和想做的事拼进10分钟里。
原理调用提示
"刻度拆解法"——把10分钟拆成10个"1分钟块"。关键：每个活动分配几个块？
建构步骤
  1. 列清单：课间必须做的（上厕所、喝水、准备书本）和想做的（和同学玩、看窗外的树）。
  2. 标时间：给每个活动标出需要的时间（精确到1分钟）。
  3. 画拼图：画10个连续方格，代表10分钟，填入活动。
  4. 调整：内容超了？去掉一个"想做的"或缩短时间。
  5. 验证：课间按方案执行一次，检查是否够用。
脚手架推荐
一张纸画10个大方格（像巧克力条）、彩色笔区分"必须做"和"想做"、计时器。
成果表达
课间10分钟"时间拼图"——10个连续格子的涂色图：蓝色格子=必须做的事，绿色格子=想做的事。每个格子标1分钟。
自评量规
  • [ ] 我列齐了课间必做的所有事
  • [ ] 我的时间拼图正好填满10个格子，不多不少
  • [ ] 我把"必须做"和"想做"用不同颜色区分了
  • [ ] 我实际执行了一次并验证方案可行
  • [ ] 我根据执行结果做了一次调整（如果有需要）
4
学科表达型
设计时间换算操练
▼
真实建构情境
你发现班上有同学总是搞混"时化分"和"分化秒"——到底是乘以60还是除以60？你想设计一个好玩的东西，让同学们玩着玩着就记住了。
建构任务
设计一款"时、分、秒换算棋"游戏的棋盘和规则。
原理调用提示
"锚定时刻比较法"——用"1时=60分"这个锚点设计起点格。"多感官验证法"——每个格子用不同方式呈现换算（看算式、读口诀、动手算）。
建构步骤
  1. 设计一个20格的棋盘，起点是"1时"，终点是"通关！"。
  2. 每格写一个换算题，如"3时=？分""120分=？时"。
  3. 设计前进和后退规则：答对前进2格，答错后退1格。
  4. 设计3张"锦囊卡"，每张写一条换算口诀（如"大化小乘60"）。
  5. 和同桌试玩一次，记下哪些题大家容易错，修改棋盘。
脚手架推荐
硬纸板画棋盘格子、骰子或硬币做棋子、换算口诀卡、彩色笔。
成果表达
"时、分、秒换算棋"——A4纸大小的棋盘，含：20个游戏格子+起点终点+3张锦囊卡+游戏规则说明。
自评量规
  • [ ] 我的棋盘至少有20个格子
  • [ ] 每个格子的换算题答案我都验算过了，100%正确
  • [ ] 我设计了前进/后退规则
  • [ ] 我制作了至少3张锦囊卡
  • [ ] 我和至少1位同学试玩并收集了反馈
5
跨学科整合型
跑步计时分析
▼
真实建构情境
体育课要测50米跑，老师说"你们的成绩精确到0.1秒"。小华跑了3次，成绩分别是9.8秒、10.2秒、9.5秒。你发现：同样是跑50米，每次时间为啥不一样？你决定用数学画图的方法来分析。
建构任务
收集自己或同学的跑步时间数据，画一张"运动时间折线图"并写出分析报告。
原理调用提示
"刻度拆解法"——把"成绩"拆成"起跑反应时间+途中跑时间"。"锚定时刻比较法"——用"最快的一次"做锚点，比较其他几次。
建构步骤
  1. 跑3~5次50米（或100米），请同学帮你记下每次的成绩（秒）。
  2. 在纸上画纵轴（时间，从0开始标到你的最慢成绩+1秒）和横轴（第几次）。
  3. 把每次成绩在图上标点，用线连起来。
  4. 分析：最快和最慢差了多少秒？你觉得为什么会有差别？
  5. 写结论：如果下次想跑更快，哪个环节需要改进？
脚手架推荐
方格纸（画折线图用）、秒表、尺子、彩色铅笔。
成果表达
运动时间折线图分析报告——一张折线图（纵轴时间/横轴次数）+ 一段分析文字（最少50字）：你最慢和最快的差距是多少秒？为什么？下次怎么改进？
自评量规
  • [ ] 我至少跑了3次并记录了真实数据
  • [ ] 我的折线图纵轴标了时间刻度，横轴标了次数
  • [ ] 我画出了折线并标注了最快和最慢的点
  • [ ] 我计算了最快和最慢的差距（秒）
  • [ ] 我写了至少50字的分析结论
6
社会情境型
公交/地铁时刻解读
▼
真实建构情境
周末你要和妈妈去图书馆，妈妈说"咱们坐公交车去，40分钟一班车，你自己算算时间"。你看了一眼站牌：上一班车是8:20到的，现在8:35——你该怎么规划出门时间？
建构任务
根据公交站牌信息，规划一个完整的出行时间方案并画成路线图。
原理调用提示
"锚定时刻比较法"——以上一班车的到达时间为锚点，推算下一班。"刻度拆解法"——把"出行"拆成"等车+坐车+走路"三段分别算时间。
建构步骤
  1. 观察公交站牌，记录发车间隔（比如"高峰期5分钟一班，平峰15分钟一班"）。
  2. 记下你到站的时间，计算下一班车预计几点到。
  3. 估算坐车+走路到目的地要多久，加总总用时。
  4. 画一张路线图，标出时间节点：几点出门→几点到站→几点上车→几点下车→几点到图书馆。
  5. 按方案执行一次，看实际时间和你算的差多少。
脚手架推荐
公交站牌照片或时刻表、纸笔画路线图、计时器、手表。
成果表达
出行时间规划路线图——一张手绘路线图，标注5个时间节点（出门→到站→上车→下车→到达）。每个节点旁写预计时间和实际时间。
自评量规
  • [ ] 我准确记录了公交发车间隔
  • [ ] 我推算出了下一班车的到达时间
  • [ ] 我的路线图标注了至少5个时间节点
  • [ ] 每个节点同时标注了"预计"和"实际"时间
  • [ ] 我计算了总用时并分析了偏差原因
7
综合创新型
设计一个时间工具
▼
真实建构情境
你已经学会了"时、分、秒"的各种知识！现在给你一个终极挑战：如果让你发明一个全新的、用来测量或展示时间的工具（不是普通的钟表），你会设计什么？这个工具可以帮那些"总是搞不清时间"的人。
建构任务
设计一个"我的神奇钟表"创意模型——画出设计图+写出功能说明书。
原理调用提示
"情境绑定法"——让时间显示和你熟悉的事情挂钩（如"该做眼保健操了"指示灯）。"多感官验证法"——你的工具至少用两种方式提醒时间（如+声音+颜色）。
建构步骤
  1. 想一个"目标用户"——谁最需要你的时间工具？（如小学生、爷爷奶奶、爱迟到的人）
  2. 确定你的工具用什么方式显示时间（颜色、声音、震动、气味？）
  3. 画设计图：正面、侧面，标出每个部件的功能。
  4. 写说明书：你的工具叫什么名字？怎么用？解决了什么问题？
  5. 如果有材料，做一个简易模型（纸盒、黏土等）。
脚手架推荐
白纸画设计图、彩色笔、废旧材料（纸盒、瓶盖、彩纸、橡皮泥）做模型。
成果表达
"我的神奇钟表"创意模型——①设计图（正面+侧面视图，标功能）；②功能说明书（名称、目标用户、使用方法、解决了什么问题）；③如有可能——实物模型。
自评量规
  • [ ] 我明确了目标用户是谁
  • [ ] 我的设计图画了至少两个角度（正面+侧面）
  • [ ] 设计图上的每个部件都标注了功能说明
  • [ ] 我的说明书写了：名称、用法、解决的问题
  • [ ] 我的工具用了至少2种感官（看+听/看+摸等）
