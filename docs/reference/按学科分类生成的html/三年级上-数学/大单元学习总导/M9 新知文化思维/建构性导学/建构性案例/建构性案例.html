个人理解型

案例一：我的"10秒钟"计时器

真实建构情境：你和同学比赛"谁估计的10秒最准"，不用钟表，完全靠感觉。你打算用什么方法让自己估计得最准？

建构任务

设计并制作一个"人体10秒计时器"方案——用拍手、脉搏、数数或其他身体节奏来估计10秒。

原理调用：原理①（从身体感受到数学概念）——先找到你身体里稳定的"节奏"（心跳、呼吸、拍手频率），再换算成秒。

建构步骤

  1. 找出身体里最稳定的节奏
  2. 实际计时验证节奏
  3. 调整校准
  4. 写出"使用说明书"
  5. 画计时器示意图

成果表达

一份"人体10秒计时器说明书"（含方法描述+校准数据+示意图）

我找到了至少两种身体节奏方法
我记录了校准数据
说明书别人能看懂
生活应用型

案例二：课间10分钟"时间管家"

真实建构情境：课间10分钟，你总是玩着玩着就上课了。请设计一个"课间10分钟时间分配方案"，让你既能玩好又不迟到。

建构任务

制作一张"课间10分钟时间管理卡"，把10分钟分成几段（如：上厕所→喝水→玩→回座位），并为每段分配时间。

原理调用：原理③（三步建构法）——先记录你实际课间做什么→再整理数据→最后制定分配方案。

建构步骤

  1. 连记3天课间时间使用
  2. 整理出"时间去哪儿了"表
  3. 设计理想方案
  4. 试用并调整

成果表达

一张"课间时间管理卡"（含时间段分配+每段任务+实际执行记录）

记录了至少3天的课间时间
分配方案总时间是10分钟
试用后做出了调整
校园问题解决型

案例三：班级"一分钟挑战大赛"

真实建构情境：班会上，老师让大家感受"一分钟能做多少事"。有人说能写30个字，有人说能跳120个绳。到底一分钟能做什么？

建构任务

设计一个"班级一分钟挑战大赛"方案——确定挑战项目、规则和记录表，让全班同学体验"一分钟的价值"。

原理调用：原理②（对比发现规律）——通过对比不同项目的比赛结果，发现不同活动中"一分钟的效率"差异。

建构步骤

  1. 列出3-5个可能的挑战项目
  2. 为每个项目制定规则
  3. 设计记录表
  4. 试运行1个项目
  5. 完善方案

成果表达

一份"班级一分钟挑战大赛策划案"（项目清单+规则+记录表模板）

至少有3个不同挑战项目
每个项目有明确的计数规则
记录表有可比性
学科表达型

案例四：时间单位换算"可视化工具"

真实建构情境："1小时=60分钟，1分钟=60秒"这个换算关系，你能不能不用死记硬背，而是用一张图让别人一眼看懂？

建构任务

创作一张"时间单位换算可视化图"，让三年级同学一看就明白"时、分、秒"之间的关系。

原理调用：原理⑤（反思+可视化）——思考"我是怎么理解这个换算关系的"，然后把理解过程画出来。

建构步骤

  1. 画出三种时间单位的"实物类比"（秒=眨一下眼，分=一节课的1/40）
  2. 画出换算阶梯或齿轮图
  3. 标注关键换算点
  4. 用彩笔上色区分层级

成果表达

一张A4大小的"时间单位换算可视化图"

包含时、分、秒三种单位
正确标注了60倍换算关系
使用了至少一种可视化表达（阶梯/齿轮/圆环）
跨学科整合型

案例五：时间与速度——测量"我走得多快"

真实建构情境：学校要开运动会，老师说"走得快的人，走同样的路用的时间少"。时间、距离、速度到底是什么关系？

建构任务

测量从教室到校门口的距离和所用时间，计算出"你的步行速度"，并画出一张"时间-距离关系图"。

原理调用：原理③（三步建构法）——先体验走路→再记录时间和距离数据→最后画图建模。

建构步骤

  1. 测量教室到校门口的步数（估算距离）
  2. 计时走这段路
  3. 计算速度（距离÷时间）
  4. 画时间-距离关系图

成果表达

一张"我的步行速度报告"（含距离、时间、速度数据+时间-距离坐标图草图）

记录了准确的时间和步数
正确计算了速度
坐标图标注了数值
社会或历史情境型

案例六：古代人的时间测量智慧

真实建构情境：古代没有钟表，古人是怎么知道时间的？日晷、水钟、沙漏——这些"原始钟表"的原理是什么？

建构任务

选择一种古代的计时工具（日晷/水钟/沙漏），用身边的材料做一个简易模型，并写一份"原理说明书"。

原理调用：原理①（从感受到概念）——古人也是通过"观察到现象→形成概念"的过程发明了计时工具。

建构步骤

  1. 查阅或回忆古人计时方式
  2. 选择一种工具研究原理
  3. 用纸杯/木棍/沙子等材料制作
  4. 测试是否存在误差
  5. 写原理说明书

成果表达

简易古代计时器模型 + "原理说明书"（含结构图+工作原理+误差分析）

模型使用了身边材料制作
说明书画出了结构
测试了计时功能并记录了误差
综合创新型

案例七：为"时间盲人"设计一个时间感知器

真实建构情境：钟表依赖视觉——如果你闭上眼睛，怎么知道时间过去了多久？或者说，如果一个人看不见也听不到，他怎样感知时间？

建构任务

设计一个"非视觉时间感知方案"——不用眼睛看钟表，用触觉、听觉、嗅觉或身体感觉来感知时间流逝。

原理调用：原理①+原理③——从身体节奏感受出发，用三步法建构出方案。

建构步骤

  1. 列出人类除视觉外的所有感官
  2. 为每种感官设计一种"时间提醒方式"
  3. 选择最佳方案画设计图
  4. 制作简易原型
  5. 测试并优化

成果表达

一份"非视觉时间感知器"设计方案（含设计图+材料清单+使用说明）

使用了至少2种非视觉感官
设计图标注了关键结构
方案有可操作性，不是空想
