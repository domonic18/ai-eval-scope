四、建构性原理

一、建构性原理提炼

同学们，建构性原理是从具体经验中提炼出来的一般方法。回顾前两步的学习——你跟着示例研究了万以内加法和减法的计算过程，又自己解决了一个新的实际问题。现在，请你思考：我在这两步中，是怎样学会新知识的？有哪些做法是有效的、可以迁移到其他学习中的？

下面，我把从这次学习经历中提炼出的5条建构性原理分享给你。

原理1：找旧知，搭桥梁

原理说明：学习新知识时，先想一想：这个问题或知识点，让我联想到了什么已经学过的内容？新知识从来不是凭空出现的，它往往站在旧知识的肩膀上。

原理来源：在学习多位数加减法竖式计算时，你发现列竖式的方法和两位数加减法完全相同，只是位数更多了。这就是"旧知"在支撑"新知"。

适用情境：学习任何新知识时，先做"旧知检索"——翻看课本或笔记，找到和新知识最相近的已学内容。比如学习乘法分配律，先回顾加法结合律；学习分数除法，先回顾分数乘法。

注意事项：不是所有旧知都能直接迁移，要仔细比较新旧知识的异同点。如果发现旧方法用不上，正好是新知识需要新思路的地方。

原理2：画出来，看得见

原理说明：遇到复杂的数量关系或计算过程，把它画成图、列成表、做成结构图。视觉化呈现能让抽象的算理变得形象，帮助你发现隐藏的规律。

原理来源：在分析多位数加减法竖式时，把相同数位对齐、标出进位退位箭头，这些可视化操作让你清楚看到"满十进一"和"不够减就借"的道理。

适用情境：理解应用题的数量关系时画线段图；整理单元知识时画思维导图；分析计算错误时画出算式步骤图；比较概念时画对比表。

注意事项：画图不是目的，帮你思考才是目的。不要为了"画得好看"而浪费时间，要专注于把思考过程展示出来。

原理3：分步骤，化难为易

原理说明：把复杂的问题拆成几个小步骤，一步一步完成。每一步完成后再检查，然后进入下一步。这样做可以降低认知负担，保证每一步都不出错。

原理来源：计算567+389时，你先把567和389对齐，然后从个位算起：7+9=16，写6进1；十位6+8+1=15，写5进1；百位5+3+1=9，结果是956。分成这三步，每一步都简单明了。

适用情境：多位数加减法竖式计算、混合运算顺序处理、复杂应用题拆解、实验操作步骤整理。

注意事项：分步骤后要记住：完成一步就回头检查一步，不要等全部做完才检查。检查的习惯比最后回头重做更有效率。

原理4：找证据，验真假

原理说明：当你有了一个想法或猜测时，想办法找证据来验证。证据可以来自计算结果、生活实例、同学的做法或自己的反向思考。证据能帮你判断想法是否正确。

原理来源：算出567+389=956后，你用956-389来验证，因为减法是加法的逆运算。如果956-389=567，就说明加法做对了。这就是用证据检验。

适用情境：验证计算结果是否正确（用逆运算或估算）；判断自己的理解是否正确（举反例）；判断某种解法是否通用（多试几个例子）。

注意事项：证据要具体，不能是"我觉得对"这样的主观感觉。要学会说"因为……所以……"来表达推理过程。

原理5：说出口，讲清楚

原理说明：学完一个新知识后，试着用自己的话把它讲一遍。能讲清楚，说明你真的理解了；讲不出来的地方，就是还需要加强的地方。

原理来源：学习完多位数加减法竖式计算后，你能用自己的话说出"为什么要相同数位对齐"、"进位是什么意思"、"竖式和横式有什么关系"，这些话就是你的建构成果。

适用情境：每学完一个知识点后做"小老师讲给自己听"；给同学讲解不会的题目；用语音备忘录录下自己的解题思路；完成学习单上的"今日收获"栏目。

注意事项：讲解时要假设对方完全不懂这个知识，像给一年级小朋友讲一样。这样你就会发现自己哪里其实还模糊。

二、建构性原理迁移应用

练习1：两位数乘一位数的计算
情境：数学书上有一道题：12×3=？老师还没有教多位数的乘法，但你想自己先想办法算出来。
建构任务：请你用已经学过的知识，自己探索出12×3的计算方法。
可用证据：
  • 12个3相加可以用加法算：3+3+3+……（加12次）
  • 可以把12分成10和2，分别乘以3，再把结果相加
  • 可以用小棒图帮忙理解
建议使用的原理：原理1（找旧知，搭桥梁）+原理3（分步骤，化难为易）
建构脚手架提示：
  • 12×3就是12个3相加，你能用加法算出结果吗？
  • 你学过整十数乘一位数，10×3=？2×3=？
  • 把12分成10和2，分别乘3再相加，这个方法叫什么？
表达：请你把自己的计算过程和思考步骤写下来，计算出12×3=36，然后用自己的话说一说：你是怎么想到这个方法的？
练习2：设计一个购书方案
情境：书店开展促销活动，《趣味数学》每本23元，《数学故事》每本18元。你有50元，想买两本不同的书。
建构任务：请你找出所有可能的购书方案，并说明每个方案需要花多少钱、还剩多少钱。
可用证据：
  • 可以用竖式计算：23+18=？
  • 可以估算：23不到25，18不到20，两本书不超过45元
  • 可以列举：先想清楚有哪几种组合，再逐个计算
建议使用的原理：原理2（画出来，看得见）+原理4（找证据，验真假）
建构脚手架提示：
  • 可以先把两种书的价钱加起来，看看50元够不够
  • 有哪几种不同的买法？把每种买法都列出来
  • 每种买法花多少钱？怎么验证你的计算没有错？
表达：请你把思考过程画成图或表格展示出来，并计算出每种方案的结果，然后说一说：你用什么方法确保不遗漏任何一种方案？

五、元认知反思

学完了建构性原理，请你想一想：

  1. 今天这5条原理中，你觉得哪一条对你帮助最大？为什么？
  2. 在解决"12×3"问题时，你有没有尝试过其他方法？后来为什么选择了现在的做法？
  3. 如果让你把这5条原理用一句话总结，你会怎么说？
  4. 这套学习方法，你可以用到其他学科吗？请举例说明。
  5. 在今天的学习过程中，你遇到过什么困难？后来是怎么解决的？
提示：元认知就是"对自己思考过程的思考"。回答这些问题能帮你发现自己是怎么学习的，为以后成为更独立的学习者打下基础。
