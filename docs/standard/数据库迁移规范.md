# 数据库迁移操作规范（Prisma）

> 适用：`web/backend` 数据库。Prisma 管理结构定义（`prisma/schema.prisma`），建库/更新由 SQL **手动执行**控制。

---

## 一、初始化全新数据库

按 `migrations/` 时间戳顺序执行 SQL + 标记已应用 + 生成 client：

```bash
cd web/backend

# 1. 按时间戳顺序执行所有 migration.sql
for d in $(ls -d prisma/migrations/*/ | sort); do
  psql "$DATABASE_URL" -f "${d}migration.sql"
done

# 2. 标记全部迁移为已应用（写 _prisma_migrations，防重复）
for d in $(ls -d prisma/migrations/*/ | sort); do
  npx prisma migrate resolve --applied "$(basename "$d")"
done

# 3. 生成 client
npx prisma generate
```

---

## 二、表结构变更（新增表/字段/索引/关系）

```bash
cd web/backend

# 1. 改 prisma/schema.prisma（加 model / 字段 / @@index 等）

# 2. 生成迁移 SQL（仅生成，不执行）
npx prisma migrate dev --create-only --name <描述性名>   # 如 add_user_avatar

# 3. review 生成的 SQL
cat prisma/migrations/*_<描述性名>/migration.sql

# 4. 手动执行 SQL
psql "$DATABASE_URL" -f prisma/migrations/*_<描述性名>/migration.sql

# 5. 标记已应用 + 生成 client
npx prisma migrate resolve --applied "$(ls prisma/migrations | tail -1)"
npx prisma generate
```

迁移命名：`<动词>_<对象>`（`add_xxx` / `drop_xxx` / `create_xxx_table` / `rename_xxx`）。

---

## 三、命令速查

| 命令 | 行为 | 自动执行 SQL |
|------|------|--------------|
| `npx prisma migrate dev --create-only --name X` | 生成 migration.sql，不执行 | ❌ |
| `npx prisma migrate resolve --applied X` | 标记迁移已手动执行 | — |
| `npx prisma generate` | 从 schema 生成 @prisma/client | — |
| `npx prisma migrate status` | 查看迁移应用状态 | — |
| `npx prisma migrate deploy` | 自动执行未应用迁移（CI/生产） | ✅ |
| `npx prisma studio` | GUI 浏览数据 | — |

---

## 四、禁止事项

- ❌ `prisma migrate dev`（无 `--create-only`）——自动执行 SQL，绕过手动控制
- ❌ `prisma db push`——绕过迁移记录，破坏 `_prisma_migrations` 一致性
- ❌ 手动 `ALTER TABLE`（不经 schema + migration）——导致 schema 与库不一致
- ❌ 手动执行 SQL 后跳过 `migrate resolve --applied`——迁移记录不一致，下次重复执行
